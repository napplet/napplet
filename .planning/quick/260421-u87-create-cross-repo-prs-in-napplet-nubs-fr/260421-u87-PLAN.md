# Quick Task 260421-u87 — Cross-repo PRs for v0.29.0 drafts

## Task

Open 4 separate pull requests in the `napplet/nubs` public repo from the
v0.29.0 draft spec files authored in Phase 135:

- `NUB-CLASS.md` — class-track root (abstract authority)
- `NUB-CLASS-1.md` — strict baseline posture
- `NUB-CLASS-2.md` — user-approved explicit-origin posture
- `NUB-CONNECT.md` — manifest-tag + CSP surface that feeds NUB-CLASS-2

Sources are archived at
`.planning/milestones/v0.29.0-phases/135-cross-repo-spec-work/drafts/`.

## Why 4 separate PRs (not grouped)

Matches strict prior precedent — every existing NUB spec (NUB-RELAY,
NUB-IDENTITY, NUB-STORAGE, NUB-IFC, NUB-THEME, NUB-KEYS, NUB-MEDIA,
NUB-NOTIFY, NUB-RESOURCE, NUB-CONFIG) is its own branch and its own PR.
The class sub-track documents cross-reference each other, but that is fine
for parallel open drafts — merge order is not a prerequisite for review.

## Execution (one cycle per draft, then batch registry fixups)

For each draft in order — `nub-class`, `nub-class-1`, `nub-class-2`,
`nub-connect` — in `/home/sandwich/Develop/nubs`:

1. `git checkout master && git pull --ff-only`
2. `git checkout -b <branch-name>`
3. Copy the draft file to repo root with its canonical name
4. Append registry row to `README.md` under "NUB-WORD (Interface Specs)"
   with placeholder PR link `https://github.com/napplet/nubs/pull/0`
5. Commit (single commit per draft — `docs: add NUB-<NAME> spec` + registry row)
6. `git push -u origin <branch>`
7. `gh pr create` with draft-status body mirroring the NUB-CONFIG PR format

After all 4 PRs open:

8. For each branch, replace the registry placeholder link with the real
   PR number and push a follow-up commit (matches the
   `fix: update NUB-XXX registry link to PR #NN` precedent from
   NUB-THEME/NUB-CONFIG)

## Constraints

- **Zero-grep hygiene:** Drafts already verified clean against
  `@napplet/`, `kehto`, `hyprgate`, `packages/(nub|shim|sdk|vite-plugin)`
  (Phase 135 Plan 04). PR bodies MUST also be clean.
- **Separate branches per draft** — mirrors precedent
- **Registry order:** Append in the same order drafts are merged upstream:
  NUB-CLASS → NUB-CLASS-1 → NUB-CLASS-2 → NUB-CONNECT (CONNECT cites
  CLASS-2 so it goes last; its registry row links CLASS-2 by name)
- **Operate only in `/home/sandwich/Develop/nubs`** — napplet repo is not
  modified by this task except for the quick-task artifacts below
- **Base branch:** `master` in all PRs
- **Draft vs ready:** Open as non-draft (matches NUB-CONFIG PR #14
  precedent; spec-complete-but-under-review is the repo's convention)
