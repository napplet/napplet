Do NOT use `import { type NappletGlobal, ... } from '@napplet/core'` mixed-form — split is clearer for a 2-symbol type group.

2. **Add a module-level default `supports()` helper** above the `(window as unknown as { napplet: NappletGlobal }).napplet = { ... }` literal (i.e., insert it right before line 137's `// ─── window.napplet global installation ──` divider, so it lives in the same section that mounts it). Name it `defaultShellSupports`. Implementation:

   ```ts
   // ─── Default shell.supports() ────────────────────────────────────────────────
   // Fallback for napplets running without a shell parent (or before a shell has
   // overwritten window.napplet.shell). Reports `true` for every NUB domain the
   // shim itself installs (bare shorthand or `nub:`-prefixed) and `false` for
   // permissions and unknown strings — the shim cannot grant permissions.
   // A shell is free to replace window.napplet.shell.supports at runtime.

   function defaultShellSupports(capability: NamespacedCapability): boolean {
     // perm:* — shell-granted only; nothing for the shim to assert.
     if (typeof capability === 'string' && capability.startsWith('perm:')) return false;

     // 'nub:<domain>' — strip the prefix and check the domain list.
     if (typeof capability === 'string' && capability.startsWith('nub:')) {
       const domain = capability.slice(4);
       return (NUB_DOMAINS as readonly string[]).includes(domain);
     }

     // Bare NUB shorthand (e.g. 'relay').
     return (NUB_DOMAINS as readonly string[]).includes(capability);
   }
   ```

   Notes:
   - The `as readonly string[]` widening on `NUB_DOMAINS` is required so `.includes()` accepts an arbitrary `string` argument without TS narrowing complaints (the typed `NubDomain[]` `includes` would only accept a `NubDomain`, defeating the runtime check).
   - The two `typeof capability === 'string'` guards are belt-and-braces against future `NamespacedCapability` widening / hostile callers passing non-strings; they cost nothing at runtime and keep the function honest.
   - `capability.slice(4)` — `'nub:'` is exactly 4 chars.
   - Do NOT add `console.warn` or any other diagnostic — this is a silent default.
   - Do NOT export this helper. It is an internal implementation detail of the shim's bootstrap; making it `export function` would invite consumers to depend on a "default" surface that should remain private.

3. **Wire the helper into the literal**, replacing the stub at lines 209-214:

   ```ts
   shell: {
     supports: defaultShellSupports,
   },
   ```

   The reference-form (`supports: defaultShellSupports`) — not a re-wrapping arrow — is intentional: it makes it cheap for a shell to detect the default by identity if it ever wants to (`window.napplet.shell.supports === defaultShellSupports` semantics), and avoids a needless extra closure. The whole `shell` namespace remains a plain object literal so a shell can still overwrite `window.napplet.shell.supports` (or `window.napplet.shell` wholesale) at runtime — no `Object.defineProperty`, no `Object.freeze`, no getter trickery.
