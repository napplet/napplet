## E2E Flows Verified

1. **Author-declared connect origins → manifest → shell grant → runtime fetch** — COMPLETE
   - `vite.config.ts` → `nip5aManifest({ connect })` → normalizeConnectOrigin → aggregateHash fold → manifest tags emitted → shell reads → shell prompts user → shell emits CSP header + `<meta name="napplet-connect-granted">` → shim reads meta → `window.napplet.connect.{granted, origins}` populated → `fetch()` works against granted origins.
2. **Shell class assignment → `window.napplet.class`** — COMPLETE
   - Shell sends `class.assigned` envelope → central dispatcher routes `class.*` prefix → `handleClassMessage` validates (integer, non-negative) → `window.napplet.class` updated.
3. **Grant revocation** — COMPLETE
   - Next load without meta → defaults `{granted: false, origins: []}` restored; shell sends `class.assigned` with `class: 1` → `window.napplet.class === 1`.
