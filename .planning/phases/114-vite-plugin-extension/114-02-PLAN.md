<objective>
Add the `validateConfigSchema()` build-time structural guard to `@napplet/vite-plugin` and wire it into the `configResolved` hook so malformed napplet config schemas fail the build early (before the manifest is generated, before `index.html` is transformed). Covers requirement VITE-07 in full.

Purpose: The Vite plugin is the build-time gatekeeper between a napplet's source schema and the NIP-5A manifest / runtime meta tag. A malformed schema that reaches the shell produces `config.schemaError` pushes at runtime — a terrible developer experience when the error is structurally detectable at build. This plan implements the four rejection rules enumerated in `114-CONTEXT.md` decisions block and the NUB-CONFIG Schema Contract / Exclusions section:

1. **Root shape**: schema root MUST be `{ type: "object", ... }`.
2. **External `$ref` ban**: any `$ref` whose value does not start with `#/` is rejected. Same-document refs are still rejected in the shell (Core Subset excludes `$ref` entirely) but for the build-time guard only the external-ref rule applies structurally — same-document refs can be flagged by the shell's full validator, and the build-time pass keeps its surface narrow.
3. **`pattern` keyword ban**: `pattern` anywhere in the schema triggers rejection (CVE-2025-69873 ReDoS class).
4. **`x-napplet-secret` + `default` ban**: a property object with both keys triggers rejection (secret-with-default).

All four violations accumulate into a single array so the error log surfaces every problem in one build cycle — developers fix once, build succeeds. No external validator dependency (no `ajv`, no `@cfworker/json-schema`, no `zod`); the guard is a ~40-60 LOC recursive walker as specified in 114-CONTEXT.md specifics.

Output: `packages/vite-plugin/src/index.ts` gains:
- `validateConfigSchema(schema: unknown): { ok: true } | { ok: false, errors: string[] }` — pure function, placed near `discoverConfigSchema` from plan 01.
- Integration into the `configResolved` async hook: after `resolvedSchema = result.schema`, if non-null, call `validateConfigSchema`; on `ok:false` throw a Vite build error with every message joined.

`pnpm --filter @napplet/vite-plugin type-check && pnpm --filter @napplet/vite-plugin build` passes clean. A fixture schema (constructed inline in the verify step) with every violation produces four rejection lines.

NON-GOALS:
- No shell-side full validator (that's a shell concern, not NUB-CONFIG spec or vite-plugin scope).
- No full Core Subset enforcement (depth limit, oneOf/anyOf/not rejection, tuple-typed array rejection). Build-time guard is the narrow four-rule band; full Core Subset enforcement is shell-side at `config.registerSchema` time.
- No manifest tag or meta tag emission (that's 114-03).
- No unit test file (repo convention: no test infra in vite-plugin — `"test:unit": "echo 'no unit tests'"` is the package.json script per 114-CONTEXT.md reference).
</objective>

### 2. Implement `validateConfigSchema`

Paste the following structural walker. The walker is recursive; every recursion carries the current schema node. A path string (dot-joined keys) is threaded so error messages locate each violation precisely.

```ts
/**
 * Structural guard for napplet config schemas at build time.
 *
 * NOT a full JSON Schema validator. Only checks the four rejection rules that
 * MUST fail the build early — full Core Subset enforcement lives in the shell
 * at `config.registerSchema` time. See NUB-CONFIG Schema Contract / Exclusions.
 *
 * Rejection rules:
 * 1. Root MUST be `{ type: "object", ... }`. Anything else -> 'invalid-schema'.
 * 2. `pattern` keyword anywhere in the tree -> 'pattern-not-allowed' (ReDoS,
 *    CVE-2025-69873 class).
 * 3. `$ref` whose value does not start with `#/` anywhere in the tree ->
 *    'ref-not-allowed' (external reference ban; same-document refs are still
 *    forbidden by the spec but are caught by the shell-side Core Subset
 *    enforcer at registerSchema time).
 * 4. Any property node where both `x-napplet-secret: true` and the `default`
 *    key are present -> 'secret-with-default'.
 *
 * Collects every violation; returns them all in one pass so the build log
 * surfaces every problem at once.
 *
 * @param schema - the unvalidated schema loaded from options / config.schema.json / napplet.config.*
 * @returns { ok: true } on pass, { ok: false, errors } on failure
 */
function validateConfigSchema(
  schema: unknown,
): { ok: true } | { ok: false; errors: string[] } {
  const errors: string[] = [];

  // Rule 1: root shape
  if (
    schema === null ||
    typeof schema !== 'object' ||
    Array.isArray(schema) ||
    (schema as Record<string, unknown>).type !== 'object'
  ) {
    errors.push(
      `invalid-schema: schema root must be { type: "object", ... } (got ${
        schema === null ? 'null' : Array.isArray(schema) ? 'array' : typeof schema === 'object' ? `type=${JSON.stringify((schema as Record<string, unknown>).type)}` : typeof schema
      })`,
    );
    // Do not recurse when root is malformed — nothing meaningful to walk.
    return { ok: false, errors };
  }

  // Rules 2-4: recursive walk
  walk(schema as Record<string, unknown>, '$', errors);

  return errors.length === 0 ? { ok: true } : { ok: false, errors };
}

/**
 * Internal: recursively walks a schema node, accumulating rule violations.
 * `path` is the dot-joined JSON-Pointer-ish path used in error messages.
 */
function walk(node: unknown, path: string, errors: string[]): void {
  if (node === null || typeof node !== 'object') return;
  if (Array.isArray(node)) {
    for (let i = 0; i < node.length; i++) {
      walk(node[i], `${path}[${i}]`, errors);
    }
    return;
  }

  const obj = node as Record<string, unknown>;

  // Rule 2: pattern keyword forbidden anywhere.
  if ('pattern' in obj) {
    errors.push(
      `pattern-not-allowed: \`pattern\` keyword found at ${path} — the Core Subset excludes \`pattern\` due to ReDoS risk (CVE-2025-69873 class). Use \`enum\`, \`minLength\`, or \`maxLength\` for constrained strings.`,
    );
  }

  // Rule 3: external $ref forbidden.
  if ('$ref' in obj) {
    const ref = obj.$ref;
    if (typeof ref !== 'string' || !ref.startsWith('#/')) {
      errors.push(
        `ref-not-allowed: \`$ref\` at ${path} must start with \`#/\` (got ${JSON.stringify(ref)}). External $ref is forbidden per NUB-CONFIG Security Considerations.`,
      );
    }
  }

  // Rule 4: x-napplet-secret:true + default coexistence forbidden.
  if (obj['x-napplet-secret'] === true && 'default' in obj) {
    errors.push(
      `secret-with-default: property at ${path} declares both \`x-napplet-secret: true\` and a \`default\` value. A secret with a hardcoded default is not a secret.`,
    );
  }

  // Recurse into child schemas. JSON Schema child-carrying keys we care about:
  //   properties: map of name -> schema
  //   items:      schema OR array of schemas (tuple form, also walked)
  //   additionalProperties: schema or boolean
  //   patternProperties: map (walked; individual `pattern` rules there are
  //     ALREADY flagged by rule 2 because the keys of patternProperties
  //     themselves are regexes — but we don't conflate the two; walking the
  //     child schemas is still correct).
  if (typeof obj.properties === 'object' && obj.properties !== null && !Array.isArray(obj.properties)) {
    for (const [key, child] of Object.entries(obj.properties as Record<string, unknown>)) {
      walk(child, `${path}.properties.${key}`, errors);
    }
  }
  if ('items' in obj) {
    walk(obj.items, `${path}.items`, errors);
  }
  if ('additionalProperties' in obj && typeof obj.additionalProperties === 'object') {
    walk(obj.additionalProperties, `${path}.additionalProperties`, errors);
  }
  if (typeof obj.patternProperties === 'object' && obj.patternProperties !== null && !Array.isArray(obj.patternProperties)) {
    for (const [key, child] of Object.entries(obj.patternProperties as Record<string, unknown>)) {
      walk(child, `${path}.patternProperties.${key}`, errors);
    }
  }
  // oneOf / anyOf / allOf / not — walked so nested pattern/ref/secret violations
  // still surface. Shell-side Core Subset enforcement rejects these combinators
  // outright at registerSchema time; the build-time guard is narrower.
  for (const combiner of ['oneOf', 'anyOf', 'allOf'] as const) {
    if (Array.isArray(obj[combiner])) {
      (obj[combiner] as unknown[]).forEach((child, i) =>
        walk(child, `${path}.${combiner}[${i}]`, errors),
      );
    }
  }
  if ('not' in obj) walk(obj.not, `${path}.not`, errors);
  // definitions / $defs — walked for the same reason
  for (const defs of ['definitions', '$defs'] as const) {
    if (typeof obj[defs] === 'object' && obj[defs] !== null && !Array.isArray(obj[defs])) {
      for (const [key, child] of Object.entries(obj[defs] as Record<string, unknown>)) {
        walk(child, `${path}.${defs}.${key}`, errors);
      }
    }
  }
}
```

```ts
async configResolved(config) {
  outDir = config.build?.outDir ?? 'dist';
  projectRoot = config.root;
  const result = await discoverConfigSchema(options, projectRoot);
  resolvedSchema = result.schema;
  resolvedSchemaSource = result.source;
  if (resolvedSchema !== null) {
    const validation = validateConfigSchema(resolvedSchema);
    if (!validation.ok) {
      const header = `[nip5a-manifest] configSchema validation failed (source: ${resolvedSchemaSource ?? 'unknown'})`;
      const body = validation.errors.map((e) => `  - ${e}`).join('\n');
      // Throw a plain Error — Vite surfaces the message and fails the build.
      // Do NOT swallow the error; malformed schemas MUST abort the build.
      throw new Error(`${header}\n${body}`);
    }
    console.log(
      `[nip5a-manifest] ${options.nappletType}: config schema discovered via ${resolvedSchemaSource} — validated`,
    );
  }
},
```
