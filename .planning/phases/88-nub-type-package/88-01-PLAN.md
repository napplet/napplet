Per NUB-01 — all 6 message types from NUB-KEYS spec:

**DOMAIN constant:**
```typescript
export const DOMAIN = 'keys' as const;
```

**Supporting types (before message types):**

`Action` — napplet-declared named action:
- `id: string` — unique action identifier, e.g. "editor.save"
- `label: string` — human-readable label for shell keybinding UI
- `defaultKey?: string` — suggested binding hint, e.g. "Ctrl+S"

`RegisterResult` — shell response to action registration:
- `actionId: string`
- `binding?: string` — key combo the shell assigned, if any

`KeyBinding` — an action-to-key mapping in the bindings list:
- `actionId: string`
- `key: string` — key combo string, e.g. "Ctrl+S"

**Base message type:**
`KeysMessage extends NappletMessage` with `type: \`keys.${string}\``

**6 concrete message interfaces (all extend KeysMessage):**

1. `KeysForwardMessage` — napplet -> shell, fire-and-forget (no id):
   - type: `'keys.forward'`
   - `key: string` (KeyboardEvent.key)
   - `code: string` (KeyboardEvent.code)
   - `ctrl: boolean`
   - `alt: boolean`
   - `shift: boolean`
   - `meta: boolean`

2. `KeysRegisterActionMessage` — napplet -> shell, correlated:
   - type: `'keys.registerAction'`
   - `id: string` (correlation ID)
   - `action: Action`

3. `KeysRegisterActionResultMessage` — shell -> napplet, correlated:
   - type: `'keys.registerAction.result'`
   - `id: string` (correlation ID matching request)
   - `actionId: string`
   - `binding?: string` (assigned key combo, if any)
   - `error?: string` (error message on failure)

4. `KeysUnregisterActionMessage` — napplet -> shell, fire-and-forget:
   - type: `'keys.unregisterAction'`
   - `actionId: string`

5. `KeysBindingsMessage` — shell -> napplet, push:
   - type: `'keys.bindings'`
   - `bindings: KeyBinding[]` (complete binding list, not diff)

6. `KeysActionMessage` — shell -> napplet, push:
   - type: `'keys.action'`
   - `actionId: string`

**Discriminated unions:**
- `KeysRequestMessage = KeysForwardMessage | KeysRegisterActionMessage | KeysUnregisterActionMessage` (napplet -> shell)
- `KeysResultMessage = KeysRegisterActionResultMessage | KeysBindingsMessage | KeysActionMessage` (shell -> napplet)
- `KeysNubMessage = KeysRequestMessage | KeysResultMessage` (all keys messages)
