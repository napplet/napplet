# Plan 13-02: Move Enforce Gate and NappKeyRegistry to Runtime

<objective>
Move the ACL enforcement gate (enforce.ts) and identity registry (napp-key-registry.ts) from @napplet/shell to @napplet/runtime, making them browser-agnostic by removing DOM dependencies (window.dispatchEvent). These are protocol-level concerns, not browser-level.
</objective>

<context>
The enforce gate (createEnforceGate, resolveCapabilities, formatDenialReason) is already pure — no DOM references. It can be moved as-is with import path changes (types from @napplet/core instead of local types.ts).

The napp-key-registry has two browser-specific calls:
1. `window.dispatchEvent(new CustomEvent('napplet:pending-update', ...))` in setPendingUpdate
2. `window.dispatchEvent(new CustomEvent('napplet:pending-update', ...))` in clearPendingUpdate

These must be replaced with the `onPendingUpdate` callback from RuntimeHooks (defined in Plan 13-01). The registry will accept a notifier function in a factory pattern instead of being a global singleton.
</context>

<task id="13-02-02" title="Move napp-key-registry to runtime as a factory">
<read_first>
- packages/shell/src/napp-key-registry.ts
- packages/runtime/src/types.ts
</read_first>
<action>
Create `packages/runtime/src/napp-key-registry.ts`. Unlike the shell version which is a global singleton with `window.dispatchEvent`, the runtime version uses a factory function `createNappKeyRegistry(notifier?)` that accepts an optional `PendingUpdateNotifier` callback.

Key changes from the shell version:
1. Import `NappKeyEntry` and `PendingUpdate` from local `./types.js` (they are defined in runtime's types.ts)
2. Import `PendingUpdateNotifier` from `./types.js`
3. Replace the singleton pattern with a factory: `export function createNappKeyRegistry(notifier?: PendingUpdateNotifier)`
4. Remove all `window.dispatchEvent(new CustomEvent(...))` calls — replace with `notifier?.(windowId)` calls
5. Remove the `typeof window !== 'undefined'` guard (no window reference at all)
6. The factory returns an object with the same method signatures as the current singleton

The returned interface:

```ts
export interface NappKeyRegistry {
  register(windowId: string, entry: NappKeyEntry): void;
  unregister(windowId: string): void;
  getPubkey(windowId: string): string | undefined;
  getEntry(pubkey: string): NappKeyEntry | undefined;
  getWindowId(pubkey: string): string | undefined;
  isRegistered(windowId: string): boolean;
  getAllEntries(): NappKeyEntry[];
  setPendingUpdate(windowId: string, update: PendingUpdate): void;
  getPendingUpdate(windowId: string): PendingUpdate | undefined;
  clearPendingUpdate(windowId: string): void;
  clear(): void;
}
```

Implementation:

```ts
export function createNappKeyRegistry(notifier?: PendingUpdateNotifier): NappKeyRegistry {
  const byWindowId = new Map<string, string>();
  const byPubkey = new Map<string, NappKeyEntry>();
  const pendingUpdates = new Map<string, PendingUpdate>();

  return {
    register(windowId: string, entry: NappKeyEntry): void {
      byWindowId.set(windowId, entry.pubkey);
      byPubkey.set(entry.pubkey, entry);
    },
    unregister(windowId: string): void {
      const pubkey = byWindowId.get(windowId);
      if (pubkey) {
        byPubkey.delete(pubkey);
        byWindowId.delete(windowId);
      }
      pendingUpdates.delete(windowId);
    },
    getPubkey(windowId: string): string | undefined {
      return byWindowId.get(windowId);
    },
    getEntry(pubkey: string): NappKeyEntry | undefined {
      return byPubkey.get(pubkey);
    },
    getWindowId(pubkey: string): string | undefined {
      return byPubkey.get(pubkey)?.windowId;
    },
    isRegistered(windowId: string): boolean {
      return byWindowId.has(windowId);
    },
    getAllEntries(): NappKeyEntry[] {
      return Array.from(byPubkey.values());
    },
    setPendingUpdate(windowId: string, update: PendingUpdate): void {
      pendingUpdates.set(windowId, update);
      notifier?.(windowId);
    },
    getPendingUpdate(windowId: string): PendingUpdate | undefined {
      return pendingUpdates.get(windowId);
    },
    clearPendingUpdate(windowId: string): void {
      pendingUpdates.delete(windowId);
      notifier?.(windowId);
    },
    clear(): void {
      byWindowId.clear();
      byPubkey.clear();
      pendingUpdates.clear();
    },
  };
}
```
