# Plan 46-02: Storage Rekeying - Remove Pubkey from Scope

## Objective

Change storage scoping from `napplet-state:{pubkey}:{dTag}:{aggregateHash}:{userKey}` to `napplet-state:{dTag}:{aggregateHash}:{userKey}`, removing the ephemeral pubkey from the storage key. Add backward-compatible dual-read migration for existing data stored under old keys. This directly fixes the "storage lost on page reload" bug since storage is now keyed by napplet type+version, not session identity.
