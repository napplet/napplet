# Plan 18-01: Move ServiceDescriptor to @napplet/core

<objective>
Add the ServiceDescriptor type to @napplet/core so all packages (shim, runtime, shell) can import it from the canonical shared location. This is the foundation type that describes a registered service — its name, version, and optional description.

This plan does NOT add semver utilities. Per CONTEXT D-07 and REQUIREMENTS CORE-03, service matching is name-only (presence/absence check). No version range parsing in this phase.
</objective>

It must be added to packages/core/src/types.ts with the identical shape:

export interface ServiceDescriptor {
  /** Unique service identifier (e.g., 'audio', 'notifications', 'clipboard'). */

  /** Semver version of the service implementation. */
  version: string;
  /** Human-readable description of the service. */
  description?: string;
