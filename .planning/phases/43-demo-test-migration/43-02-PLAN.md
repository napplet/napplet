# Plan 43-02: Migrate Test Fixtures and Update E2E Config

## Objective

Update the `publish-napplet` test fixture from named shim import to raw `window.napplet.*` globals (per CONTEXT decision D-03: test fixtures test protocol behavior, not developer ergonomics). Update the e2e vite config to add the `@napplet/sdk` resolve alias so demo napplets resolve correctly when served during e2e tests.

## Tasks

<task id="43-02-01">
<title>Migrate publish-napplet fixture to window.napplet.relay.publish()</title>

<read_first>
- tests/fixtures/napplets/publish-napplet/src/main.ts
- .planning/phases/43-demo-test-migration/43-CONTEXT.md
- .planning/phases/41-shim-restructure/41-CONTEXT.md
</read_first>

<action>
Replace the import line in `tests/fixtures/napplets/publish-napplet/src/main.ts`:

**Remove:**
```typescript
import { publish } from '@napplet/shim';
```

**Replace with:**
```typescript
import '@napplet/shim';
```

Then update the single `publish()` call site inside the message handler:

**Old:**
```typescript
await publish({
  kind,
  content,
  tags: [],
  created_at: Math.floor(Date.now() / 1000),
});
```

**New:**
```typescript
await window.napplet.relay.publish({
  kind,
  content,
  tags: [],
  created_at: Math.floor(Date.now() / 1000),
});
```

No other changes needed. The `auth-napplet` fixture is already side-effect only and requires no changes. The `test-napplet.html` is already side-effect only and requires no changes.
</action>

<acceptance_criteria>
- `tests/fixtures/napplets/publish-napplet/src/main.ts` contains `import '@napplet/shim';`
- `grep -c "from '@napplet/shim'" tests/fixtures/napplets/publish-napplet/src/main.ts` returns 0 (no named imports)
- `grep "window\.napplet\.relay\.publish" tests/fixtures/napplets/publish-napplet/src/main.ts` returns at least 1 match
- `grep -c "^import { publish" tests/fixtures/napplets/publish-napplet/src/main.ts` returns 0
</acceptance_criteria>
</task>

<task id="43-02-02">
<title>Add @napplet/sdk alias to e2e vite config</title>

<read_first>
- tests/e2e/vite.config.ts
</read_first>

<action>
Add `@napplet/sdk` resolve alias to `tests/e2e/vite.config.ts` following the existing `@napplet/shim` alias pattern.

**Current resolve.alias block:**
```typescript
alias: {
  '@napplet/shell': path.resolve(__dirname, '../../packages/shell/src/index.ts'),
  '@napplet/shim': path.resolve(__dirname, '../../packages/shim/src/index.ts'),
},
```

**New resolve.alias block:**
```typescript
alias: {
  '@napplet/shell': path.resolve(__dirname, '../../packages/shell/src/index.ts'),
  '@napplet/shim': path.resolve(__dirname, '../../packages/shim/src/index.ts'),
  '@napplet/sdk': path.resolve(__dirname, '../../packages/sdk/src/index.ts'),
},
```

The path `../../packages/sdk/src/index.ts` is correct because the e2e config is at `tests/e2e/` (2 levels deep from repo root).
</action>

<acceptance_criteria>

</acceptance_criteria>
</task>
