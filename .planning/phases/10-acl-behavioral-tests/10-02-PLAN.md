  ### TST-04: Mid-Session Revoke Tests

  These tests prove that revoking a capability takes effect immediately — the very next message of that type is denied. No stale grants.

  1. `TST-04: mid-session relay:write revoke — publish blocked after revoke`:
     - Publish a kind 1 event. Wait for OK success. Assert success.
     - Clear messages.
     - Revoke relay:write on the nappEntry.
     - Publish another kind 1 event (different eventId). Wait for OK. Assert `parsed.success === false`. Assert reason contains `'relay:write'`.
     - This proves the revoke took effect immediately with no stale grant.

  2. `TST-04: mid-session relay:read revoke — subscription delivery blocked after revoke`:
     - Load TWO auth napplets (sender and receiver).
     - Receiver creates subscription for bus kind 29003 with topic filter `#t: ['lifecycle-test']`.
     - Sender publishes kind 29003 event with tag `['t', 'lifecycle-test']`.
     - Wait for EVENT delivery to receiver (poll for EVENT with receiver's subId).
     - Clear messages.
     - Revoke relay:read on receiver's nappEntry.
     - Sender publishes another kind 29003 event with same topic (different eventId and created_at = now).
     - Wait 1500ms.
     - Verify NO EVENT delivery to receiver's subscription (check that no EVENT messages with receiver's subId appeared after clear).
     - This proves delivery-time relay:read check blocks immediately after revoke.

  3. `TST-04: mid-session state:write revoke — state-set blocked after revoke`:
     - Send state-set request with key='test-key', value='first', corrId='corr-mid-01'. Wait for response. Assert no error (write succeeds).
     - Clear messages.
     - Revoke state:write on nappEntry.
     - Send state-set request with key='test-key', value='second', corrId='corr-mid-02'. Wait for response. Assert error contains `'state:write'`.
     - This proves state:write revoke is immediate.

  4. `TST-04: mid-session sign:event revoke — signer request blocked after revoke`:
     - Configure mock signer: `__setSigner__({ getPublicKey: () => 'a'.repeat(64), signEvent: (e) => Promise.resolve(e), getRelays: () => ({}) })`
     - Send kind 29001 signer request with method='getPublicKey'. Wait for OK. Assert success.
     - Clear messages.
     - Revoke sign:event on nappEntry.
     - Send another signer request with method='getPublicKey' (new eventId). Wait for OK. Assert `parsed.success === false`. Assert reason contains `'sign:event'`.

  5. `TST-04: mid-session block — all operations blocked immediately`:
     - Publish event. Assert success.
     - Clear messages.
     - Block the nappEntry.
     - Publish event. Assert denial.
     - Create subscription. Assert CLOSED with denial.
     - This proves block takes effect immediately on all operation types.

  ### TST-05: Revoke ALL Capabilities — Zero Messages

  6. `TST-05: revoke ALL capabilities — zero messages delivered to napplet`:
     - Revoke every capability individually on the nappEntry. The capabilities to revoke are: `'relay:read', 'relay:write', 'sign:event', 'sign:nip04', 'sign:nip44', 'state:read', 'state:write', 'hotkey:forward', 'cache:read', 'cache:write'` (all 10).
     - Implementation:
     ```typescript
     const allCaps = [
       'relay:read', 'relay:write', 'cache:read', 'cache:write',
       'hotkey:forward', 'sign:event', 'sign:nip04', 'sign:nip44',
       'state:read', 'state:write'
     ];
     for (const cap of allCaps) {
       await page.evaluate(
         ([p, d, h, c]) => (window as any).__aclRevoke__(p, d, h, c),
         [pubkey, dTag, aggregateHash, cap]
       );
     }
     ```
     - Clear messages.
     - Attempt publish (kind 1 event). Wait for OK false (relay:write denied).
     - Attempt subscribe (REQ). Wait for CLOSED (relay:read denied).
     - Attempt state-get (kind 29003 shell:state-get). Wait for OK false (relay:write denied — can't even reach state handler).
     - Wait 1000ms.
     - Count all shell->napplet messages that are NOT denial responses. Assert count is 0. (The only messages should be denial responses like OK false and CLOSED.)
     - Verify with aclCheck that every capability returns false:
     ```typescript
     for (const cap of allCaps) {
       const check = await page.evaluate(
         ([p, d, h, c]) => (window as any).__aclCheck__(p, d, h, c),
         [pubkey, dTag, aggregateHash, cap]
       );
       expect(check).toBe(false);
     }
     ```

  7. `TST-05: revoke ALL then re-grant relay:write — only publish works`:
     - Revoke all capabilities (same as test 6).
     - Re-grant only relay:write.
     - Publish event. Assert OK success (relay:write re-granted).
     - Create subscription. Assert CLOSED denial (relay:read still revoked).
     - This proves granular re-grant after total revoke.

  ### TST-06: ACL State Persistence

  8. `TST-06: revoke persists across simulated reload`:
     - Revoke relay:write on nappEntry.
     - Verify check returns false.
     - Persist: `__aclPersist__()`.
     - Clear ACL store (simulates reload): `__aclClear__()`.
     - Verify check returns true (permissive default after clear).
     - Restore localStorage value and load:
     ```typescript
     await page.evaluate(() => (window as any).__aclLoad__());
     ```
     - Verify check returns false (revoke restored from persistence).

  9. `TST-06: block persists across simulated reload`:
     - Block the nappEntry.
     - Verify isBlocked returns true:
     ```typescript
     const blocked = await page.evaluate(
       ([p, d, h]) => {
         const entry = (window as any).__aclGetEntry__(p, d, h);
         return entry?.blocked;
       },
       [pubkey, dTag, aggregateHash]
     );
     expect(blocked).toBe(true);
     ```
     - Persist to localStorage.
     - Capture the raw localStorage value for `'napplet:acl'`.
     - Clear ACL store.
     - Restore localStorage value.
     - Load from localStorage.
     - Verify isBlocked is still true.

  10. `TST-06: multiple capability revokes persist correctly`:
      - Revoke relay:write and state:write.

      - Verify state:write check returns false.

  11. `TST-06: persistence round-trip with behavioral verification`:
