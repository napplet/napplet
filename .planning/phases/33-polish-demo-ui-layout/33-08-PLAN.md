Purpose: Currently, wireNodeSelection() calls stopPropagation() for all clicks on node elements, which blocks service button click handlers (in the document listener) from executing. Adding a check for button clicks allows button handlers to execute, while node selection still works for non-button areas.

Output: wireNodeSelection() updated with early return guard for buttons, enabling service button handlers to execute before propagation is stopped.

Expected result (CORRECT):
```typescript
function wireNodeSelection(): void {
  const allNodes = document.querySelectorAll('[data-node-id]');
  for (const el of allNodes) {
    el.addEventListener('click', (event) => {
      // Guard: Skip button clicks to allow button handlers to execute
      if ((event.target as HTMLElement).closest('button')) return;
      
      event.stopPropagation();
      const nodeId = el.getAttribute('data-node-id');
      if (nodeId) {
        setSelectedNodeId(nodeId);
        // ... rest of node selection logic
      }
    });
  }
}
```

Implementation:
- Add this guard as the FIRST statement inside the click event listener (before stopPropagation()):
  ```typescript
  if ((event.target as HTMLElement).closest('button')) return;
  ```
- This allows button clicks to bubble up to document-level button handlers
- Node selection still fires for non-button clicks on the node card
- Keep all other wireNodeSelection() logic unchanged (lines 529-542)

Why this works:
- Button handlers are registered on `document` and listen for specific button selectors
- When a button is clicked, the event bubbles up from the button → node card → document
- With the guard, the node card listener returns early for buttons
- The event continues bubbling to the document listener, which executes the button handler
- For non-button clicks on the node card, stopPropagation() prevents further bubbling, isolating node selection
