# Plan 12-02: Rewire @napplet/shell to Import from @napplet/core

<objective>
Replace all local protocol type and constant definitions in @napplet/shell with imports from @napplet/core. Delete the duplicated definitions from shell's types.ts. Shell's types.ts retains only shell-specific types (hook interfaces, registry types). Shell's topics.ts is deleted entirely (re-exports from core instead). The shell package declares @napplet/core as a dependency.
</objective>

<context>
After Plan 12-01, @napplet/core exists and exports: NostrEvent, NostrFilter, Capability, ALL_CAPABILITIES, BusKind, BusKindValue, AUTH_KIND, SHELL_BRIDGE_URI, PROTOCOL_VERSION, REPLAY_WINDOW_SECONDS, DESTRUCTIVE_KINDS, TOPICS, TopicKey, TopicValue.

Currently, packages/shell/src/types.ts contains both protocol types (to be removed) and shell-specific types (to be kept). The protocol types section must be deleted and replaced with re-exports from @napplet/core.

Files that import from shell's types.ts or topics.ts must be updated to import protocol items from @napplet/core instead.

The shell package's index.ts must re-export protocol types from @napplet/core (for backwards compatibility — existing consumers who import from '@napplet/shell' should still find these types).
</context>

<action>
Edit `packages/shell/package.json` to add `@napplet/core` as a regular dependency:

Add to the `dependencies` field (create it if it doesn't exist, placing it before `devDependencies`):

```json

```

The `"workspace:*"` protocol tells pnpm to resolve to the local workspace package. This ensures turborepo builds @napplet/core before @napplet/shell (via the `"dependsOn": ["^build"]` task config in turbo.json).
</action>
