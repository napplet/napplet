## Tasks

### Task 1: Wire Tap Events to `recordSignerRequest()`

In `main.ts`, subscribe to the message tap to detect signer-request protocol traffic and record it:

```ts
import { recordSignerRequest } from './signer-connection.js';

tap.onMessage((msg) => {
  // Detect signer request (kind 29001, topic = signer-request direction)
  if (
    msg.verb === 'EVENT' &&
    msg.parsed.eventKind === 29001 &&
    msg.parsed.topic === undefined &&
    msg.direction === 'napplet->shell'
  ) {
    // Extract method from the raw event tags if available
    const raw = msg.raw;
    const event = (Array.isArray(raw) && raw.length > 1)
      ? (raw[1] as Record<string, unknown>)
      : null;
    const tags = (event?.tags as string[][] | undefined) ?? [];
    const method = tags.find((t) => t[0] === 'method')?.[1] ?? 'unknown';
    const eventTag = tags.find((t) => t[0] === 'event')?.[1];
    let kind: number | undefined;
    if (method === 'signEvent' && eventTag) {
      try {
        kind = (JSON.parse(eventTag) as { kind?: number }).kind;
      } catch { /* ignore */ }
    }

    recordSignerRequest({
      timestamp: msg.timestamp,
      method,
      kind,
      success: true, // preliminary; updated on OK response if available
    });
  }

  // Detect signer response failure (OK false after a signer request)
  if (
    msg.verb === 'OK' &&
    msg.parsed.success === false &&
    msg.direction === 'shell->napplet'
  ) {
    // Mark the most recent request as failed if it was within 5s
    // (lightweight heuristic since correlating by eventId requires more state)
    const state = getSignerConnectionState();
    const last = state.recentRequests[state.recentRequests.length - 1];
    if (last && Date.now() - last.timestamp < 5000) {
      recordSignerRequest({ ...last, success: false });
    }
  }
});
```

Note: The heuristic for marking failures is intentionally simple for Phase 31. A more precise correlation by event ID can be added later if needed.

### Task 2: Render Recent Activity in the Signer Node Compact Summary

Update `updateSignerNodeDisplay()` in `main.ts` (or extract to `signer-node.ts`) to render recent requests after connection:

```html
<!-- Connected state: compact summary with recent activity -->
<div class="topology-node-kicker">service</div>
<div class="topology-node-title">signer</div>
<div class="topology-node-meta signer-status-connected">
  <span class="signer-method-badge">{method}</span>
  <span class="signer-pubkey">{truncated pubkey}</span>
  {relay ? `<span class="signer-relay">{relay}</span>` : ''}
</div>
<div class="signer-recent-requests">
  <div class="signer-recent-label">recent</div>
  {recentRequests.slice(-5).reverse().map(r => `
    <div class="signer-request-row ${r.success ? 'ok' : 'err'}">
      <span class="signer-req-method">${r.method}</span>
      ${r.kind !== undefined ? `<span class="signer-req-kind">k${r.kind}</span>` : ''}
      <span class="signer-req-status">${r.success ? '✓' : '✗'}</span>
    </div>
  `).join('')}
  ${recentRequests.length === 0 ? '<div class="signer-no-requests">no requests yet</div>' : ''}
</div>
```

Show at most the last 5 requests in the compact node summary. The full 20-record history is available in the inspector.

### Task 3: Inspector Panel Integration (Phase 29 Surface)

If Phase 29's inspector panel infrastructure is available by the time Phase 31 executes, the signer service node should expose a richer inspector section.

Add a detail provider function in `signer-connection.ts`:

```ts
export interface SignerInspectorDetail {
  method: SignerConnectionMethod;
  pubkey: string | null;
  relay: string | null;
  recentRequests: SignerRequestRecord[];
  isConnecting: boolean;
  error: string | null;
}

export function getSignerInspectorDetail(): SignerInspectorDetail {
  return { ...getSignerConnectionState() };
}
```

In the inspector panel rendering code (wherever Phase 29 adds `renderInspectorFor(nodeId)`), add a case for signer service nodes that calls `getSignerInspectorDetail()` and renders:

- **Identity section:** connection method badge, full pubkey, active relay URL (NIP-46 only)
- **Activity section:** full 20-record `recentRequests` list with method, kind, success/fail, and timestamp (formatted as relative time, e.g., "3s ago")
- **Actions:** "Disconnect" button that calls `disconnectSigner()`

If Phase 29 is not yet implemented at execution time of Phase 31, this task is additive and non-blocking. The signer node still renders its compact summary independently. Mark the inspector section as a progressive enhancement.

### Task 4: Signer Node "Disconnect" Action

Add a disconnect control to the signer node so users can re-enter the connect flow:

In the connected state HTML (from Task 2), include:

```html
<button class="signer-disconnect-btn" data-action="disconnect-signer">disconnect</button>
```

In `main.ts` (or `signer-node.ts`) event delegation:

```ts
document.addEventListener('click', (e) => {
  const target = e.target as HTMLElement;
  if (target.closest('[data-action="disconnect-signer"]')) {
    disconnectSigner();
    debuggerEl?.addSystemMessage('signer disconnected');
  }
});
```

After disconnect, the signer node reverts to "Not connected" with the "Connect Signer" button visible.
