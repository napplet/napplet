<objective>
Fix signer node to flash amber on infrastructure failures

**Problem:** When a signing request fails (e.g., no signer configured), the Signer service node stays green instead of flashing amber. The Shell, ACL, and Runtime nodes correctly cascade to amber, but the Signer node is missing from the highlight path.

**Root cause:** The detectServiceTarget() function at lines 52-67 only detects SIGNER_REQUEST (kind 29001) and SIGNER_RESPONSE (kind 29002) messages. When a signer request fails, the OK response message with reason `"error: no signer configured"` does NOT have an eventKind field, so the Signer node is never added to the highlight path.

**Solution:** Add detection for signer-specific error messages in the OK response. When msg.verb is 'OK' and the reason includes 'signer', classify it as a signer target so the Signer node flashes in the highlight path.

**Output:** flow-animator.ts with enhanced detectServiceTarget() that catches signer errors via OK message reason patterns
</objective>

## Message Flow for Signer Errors

When no signer is configured:
1. Napplet sends SIGNER_REQUEST (kind 29001)
2. Signer service receives it, recognizes no signer available
3. Signer service sends OK response: `['OK', eventId, false, 'error: no signer configured']`
4. This OK message has verb='OK' but NO eventKind field
5. Current detectServiceTarget only checks eventKind, so Signer node is NOT highlighted

Solution: Detect signer errors by reason string pattern in OK responses.

## Error Message Patterns

Signer-related OK responses include:
- `"error: no signer configured"` → should highlight Signer node
- Any message with `"signer"` keyword in reason → should highlight Signer node (if signer service exists)

This approach is non-invasive (doesn't modify signer-service contract) and pragmatic for demo visualization.

```typescript
function detectServiceTarget(topology: DemoTopology, msg: TappedMessage): string | null {
  // Original detection for SIGNER_REQUEST/SIGNER_RESPONSE
  if (
    (msg.parsed.eventKind === BusKind.SIGNER_REQUEST || msg.parsed.eventKind === BusKind.SIGNER_RESPONSE) &&
    topology.services.includes('signer')
  ) {
    return 'signer';
  }

  // NEW: Detect signer errors via OK response with signer-related reason
  if (
    msg.verb === 'OK' &&
    typeof msg.parsed.reason === 'string' &&
    (msg.parsed.reason.includes('no signer') || msg.parsed.reason.includes('signer')) &&
    topology.services.includes('signer')
  ) {
    return 'signer';
  }

  // Notifications detection
  if (
    typeof msg.parsed.topic === 'string' &&
    msg.parsed.topic.startsWith('notifications:') &&
    topology.services.includes('notifications')
  ) {
    return 'notifications';
  }
  return null;
}
```
