# Plan 22.1-04: Runtime Migration — Service Dispatch for Signer, Relay, Cache

<objective>
Wire the runtime to dispatch signer requests, REQ/CLOSE, and EVENT publishing to registered
services when available, while preserving the existing internal code paths as fallbacks.
This is the incremental migration (D-01): both paths work, existing tests stay green,
new tests verify service-based routing.

CRITICAL: This is a dual-path migration bridge. The runtime checks for registered services
first, falls back to internal hooks if no service is registered. Old RuntimeHooks.relayPool
and RuntimeHooks.cache become optional. Existing tests that provide hooks directly still work.
New tests that register services verify the service path.

After this plan, shell implementors can choose: provide RuntimeHooks (old way) or register
services (new way). Both work. The old way will be deprecated in a follow-up phase.
</objective>

### Dual-path dispatch logic

**Signer (handleEvent, kind 29001 case):**
```typescript
case BusKind.SIGNER_REQUEST:
  // Service path: check for registered 'signer' service
  if (serviceRegistry['signer']) {
    serviceRegistry['signer'].handleMessage(
      windowId,
      ['EVENT', event],
      (msg) => hooks.sendToNapplet(windowId, msg),
    );
    return;
  }
  // Fallback: use internal handleSignerRequest
  handleSignerRequest(event, windowId, pubkey);
  return;
```

**Relay (handleReq, non-bus-kind):**
```typescript
// Service path: check for registered 'relay' or separate 'relay-pool'+'cache'
const relayService = serviceRegistry['relay'] ?? serviceRegistry['relay-pool'];
if (relayService) {
  relayService.handleMessage(windowId, msg, (m) => hooks.sendToNapplet(windowId, m));
  // If separate cache service exists, also query it
  if (!serviceRegistry['relay'] && serviceRegistry['cache']) {
    serviceRegistry['cache'].handleMessage(windowId, msg, (m) => hooks.sendToNapplet(windowId, m));
  }
  return;
}
// Fallback: use internal relay pool + cache hooks
// (existing code unchanged)
```

**Close (handleClose):**
```typescript
const relayService = serviceRegistry['relay'] ?? serviceRegistry['relay-pool'];
if (relayService) {
  relayService.handleMessage(windowId, ['CLOSE', subId], (m) => hooks.sendToNapplet(windowId, m));
}
// Also do internal cleanup (subscription map, hook untrack) for backwards compat
```
