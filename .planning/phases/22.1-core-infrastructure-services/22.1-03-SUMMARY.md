# Summary: Plan 22.1-03 — Coordinated Relay Helper

## What Was Done

Created `createCoordinatedRelay()` — a composite ServiceHandler in `@napplet/services` that combines relay pool and cache into a single service. This is the primary integration path for shell implementors who want coordinated relay + cache behavior out of the box.

Key behaviors:
- **REQ**: Queries cache and subscribes to relay pool in parallel. Events deduplicated by ID (seenIds Set). Unified EOSE sent only after both sources complete. 15s (configurable) fallback timer.
- **EVENT**: Publishes to relay pool AND stores in cache.
- **CLOSE**: Cancels relay pool subscription, cleans up tracked state.
- Edge cases: handles cache unavailable, relay unavailable, both unavailable.
- Relay events are stored in cache for future queries.

Registers as service name `relay` (not `relay-pool` or `cache`).
