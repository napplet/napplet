**Body — write these sections in order:**

1. **Overview** paragraph: The shell (`@napplet/shell`) acts as a NIP-01 relay for sandboxed napplet iframes. It routes postMessage traffic, enforces ACL, proxies signing and storage requests, and manages the NIP-42 AUTH handshake. The host application provides relay pool, signer, and crypto hooks via dependency injection. This skill produces a working `createShellBridge()` integration with minimum viable hooks.

2. **Prerequisites** section:
   - Node.js 18+, pnpm
   - `nostr-tools` >=2.23.3 available in the host project
   - A Nostr relay pool (e.g., `@nostr-dev-kit/ndk`, `nostr-tools/pool`, or custom)
   - A NIP-07 compatible signer (e.g., `window.nostr`, NDK signer, or test signer)

4. **Step 2 — Implement minimum viable hooks** — show the complete required `ShellHooks` object. The type is `ShellHooks` from `packages/shell/src/types.ts`. Show a realistic stub implementation with comments marking required vs optional groups:

import { createShellBridge, type ShellHooks } from '@napplet/shell';
import { verifyEvent as _verifyEvent } from 'nostr-tools/pure';

const subscriptions = new Map<string, () => void>();

const hooks: ShellHooks = {
  relayPool: {
    getRelayPool: () => myNdkInstance,          // wrap your relay pool here
    trackSubscription: (key, cleanup) => {
      subscriptions.set(key, cleanup);
    },
    untrackSubscription: (key) => {
      subscriptions.get(key)?.();
      subscriptions.delete(key);
    },
    openScopedRelay: (windowId, relayUrl, subId, filters, sourceWindow) => {
      // implement for NIP-29 group relay support — omit to disable
    },
    closeScopedRelay: (windowId) => {
      // close NIP-29 scoped relay
    },
    publishToScopedRelay: (windowId, event) => false,  // stub
    selectRelayTier: (filters) => [],                   // stub — return relay URLs
    isAvailable: () => true,
  },

  auth: {

  },

  crypto: {
    verifyEvent: async (event) => _verifyEvent(event),
    randomUUID: () => crypto.randomUUID(),

  windowManager: {

5. **Step 3 — Create the bridge and wire message listener**:
```ts
const bridge = createShellBridge(hooks);

// Wire the global message listener — all napplet postMessage traffic routes here
window.addEventListener('message', bridge.handleMessage);
```

6. **Step 4 — Register a napplet iframe** — show importing `originRegistry` from `@napplet/shell` and calling `register()` when an iframe loads. The `windowId` is an arbitrary string the host assigns. Call `sendChallenge` after registration to initiate NIP-42 AUTH:

```ts
import { originRegistry } from '@napplet/shell';

function onIframeLoad(iframe: HTMLIFrameElement, windowId: string): void {
  if (!iframe.contentWindow) return;
  // Register BEFORE sendChallenge — messages arriving before registration are dropped
  originRegistry.register(iframe.contentWindow, windowId);
  bridge.sendChallenge(windowId);
}

function onIframeUnload(windowId: string): void {
  originRegistry.unregister(windowId);
}
```

7. **Step 5 — Handle consent for destructive signing** — register a handler for kinds 0 (metadata), 3 (contacts), 5 (relay list), 10002 (NIP-46 relay list). The `resolve(boolean)` callback approves or denies:

```ts
bridge.registerConsentHandler((request) => {
  const { event, pubkey, resolve } = request;
  const allowed = confirm(
    `Napplet (${pubkey.slice(0, 8)}...) wants to sign kind ${event.kind}. Allow?`
  );
  resolve(allowed);
});
```

8. **Step 6 — Register a service (optional)** — show using `bridge.runtime.registerService()` after creation. Point to `skills/add-service/SKILL.md` for full service implementation:

```ts
import { createAudioService } from '@napplet/services';

bridge.runtime.registerService('audio', createAudioService({
  onChange: (sources) => updateAudioUI(sources),
}));
```

Services can also be provided in `hooks.services` at construction time (static wiring).

9. **Step 7 — Teardown**:
```ts
// Remove listener BEFORE destroying — destroy() does not remove it
window.removeEventListener('message', bridge.handleMessage);
bridge.destroy();
```

10. **Common pitfalls** section with at least these items:
    - `originRegistry.register()` MUST be called before `bridge.sendChallenge()`. Messages from an unregistered window are dropped silently.
    - `relayPool.getRelayPool()` must implement `RelayPoolLike` (subscription/publish/request methods) — not the raw NDK/nostr-tools pool. Wrap it with an adapter.
    - `bridge.handleMessage` is already bound — pass it directly to `addEventListener`. Do not wrap it in another arrow function.
    - `crypto.verifyEvent` must return `Promise<boolean>`. Wrap synchronous verifiers: `async (e) => _verifyEvent(e)`.
    - `bridge.destroy()` does NOT remove the `window.message` listener. Always remove it manually first.
    - Consent handler fires for kinds 0, 3, 5, 10002 only. All other signing kinds pass automatically unless the napplet's ACL blocks `sign:event`.
    - After `bridge.destroy()`, do NOT call any bridge methods. Create a new bridge instance instead.

All code blocks must be TypeScript. Use realistic examples (not abstract placeholders). No framework dependencies (no Svelte/React/Vue).
