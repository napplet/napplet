  <action>
Inspect `packages/shim/dist/index.d.ts` to confirm:
1. It contains `declare global` with `interface Window { napplet: NappletGlobal }`
2. It does NOT contain any `export declare function`, `export declare const`, or `export type { ... }` for named values/types
3. The only exports are the global augmentation (no named module exports)

Read the file and confirm the shape. The file should look roughly like:
```typescript
// (import statements from @napplet/core, etc.)
declare global {
  interface Window {
    napplet: NappletGlobal;
  }
}
export {};  // (TypeScript may emit this to mark it as a module)
```

If `export {}` is present, that is fine — TypeScript emits it automatically to mark the file as a module when there are no other exports. It does NOT represent a named export.

If named exports like `export declare function emit(...)` are found, that means the `export` keyword was not fully removed from index.ts in Plan 01. Fix accordingly.
  </action>
  <verify>
    <automated>cd /home/sandwich/Develop/napplet && echo "=== Window augmentation ===" && grep 'interface Window' packages/shim/dist/index.d.ts && echo "=== Named exports (should be 0) ===" && grep '^export declare\|^export type\|^export {' packages/shim/dist/index.d.ts | grep -v 'export {}' || echo "PASS: no named exports found"</automated>
  </verify>
  <acceptance_criteria>
