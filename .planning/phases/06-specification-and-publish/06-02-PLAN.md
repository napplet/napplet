<task type="auto">

  <files>packages/shim/package.json, packages/shell/package.json, packages/vite-plugin/package.json</files>

  <action>
  Run publint on each package and capture output:

  ```bash
  npx publint ./packages/shim
  npx publint ./packages/shell
  npx publint ./packages/vite-plugin
  ```

  Common publint warnings/errors and fixes:

  1. **"main" field points to ESM file**: If publint warns about `"main"` pointing to an ESM file when `"type": "module"` is set, this is typically just a suggestion. The `main` field is redundant when `exports` is present. Options:
     - Remove `"main"` and `"module"` fields (exports map is sufficient for modern tooling)
     - OR keep them for backwards compatibility

  2. **Missing "sideEffects" field**: Add `"sideEffects": false` to each package.json for tree-shaking optimization. This is a warning, not an error.

  3. **"module" field is non-standard**: The `module` field is bundler-only and redundant with exports. If publint flags it, remove it.

  For each error/warning, apply the fix and re-run publint until all three packages show zero errors.

  Recommended package.json structure after fixes (for each package):
  ```json
  {
    "type": "module",
    "exports": {
      ".": {
        "types": "./dist/index.d.ts",
        "import": "./dist/index.js"
      }
    },
    "types": "./dist/index.d.ts",
    "files": ["dist"],
    "sideEffects": false
  }
  ```

  Remove `"main"` and `"module"` fields if publint advises. Keep `"types"` at top level for older TypeScript versions that don't read exports conditions.
  </action>
  <verify>npx publint ./packages/shim 2>&1 | tail -1 && npx publint ./packages/shell 2>&1 | tail -1 && npx publint ./packages/vite-plugin 2>&1 | tail -1</verify>

    - Each package.json has `"sideEffects": false`

  Common arethetypeswrong issues and fixes:

  1. **"FalseESM"**: Types say CJS but package exports ESM. Fix: ensure `"type": "module"` in package.json.

  3. **"MissingExport"**: TypeScript can't resolve the export. Fix: ensure exports map has `"types"` condition.

  For each issue, apply the fix and re-run. The goal is zero problems across all resolution modes (node10, node16, bundler).
