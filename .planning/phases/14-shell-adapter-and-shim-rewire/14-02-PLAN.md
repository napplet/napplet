## Objective

Switch `@napplet/shim` to import all protocol types and constants from `@napplet/core` instead of maintaining duplicate local definitions. Delete or reduce `packages/shim/src/types.ts` to contain only shim-specific types (if any). Ensure zero behavioral changes — the shim's public API (subscribe, publish, query, emit, on, nappStorage) must remain identical.

## Context

Currently `packages/shim/src/types.ts` defines:
- `NostrEvent` interface (duplicate of shell's and core's)
- `NostrFilter` interface (duplicate)
- `BusKind` constant object (duplicate)
- `BusKindValue` type (duplicate)
- `AUTH_KIND` constant (duplicate)
- `SHELL_BRIDGE_URI` constant (duplicate)
- `PROTOCOL_VERSION` constant (duplicate)

After Phase 12, all of these exist in `@napplet/core`. The shim should import from core, eliminating the maintenance burden of keeping duplicates in sync.

The shim has NO dependency on `@napplet/runtime` or `@napplet/shell` — it only needs `@napplet/core` for types and constants, plus `nostr-tools` for event signing.
