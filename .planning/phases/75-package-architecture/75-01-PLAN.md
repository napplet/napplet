From specs/NIP-5D.md (Wire Format section):
```
All messages between napplet and shell are JSON objects with a `type` field:
    { "type": "<domain>.<action>", ...payload }
The `type` field is a string discriminant in `domain.action` format.
Domains correspond to NUB capability names.
```

From specs/NIP-5D.md (NUB Negotiation section):
```
["requires", "<nub-name>"]
Each `requires` value is a short NUB name matching a NUB domain (e.g., `relay`).
```

From specs/NIP-5D.md (Runtime Capability Query section):
```
window.napplet.shell.supports('foo')       // NUB capability -- boolean
window.napplet.shell.supports('popups')    // sandbox permission -- boolean
```
