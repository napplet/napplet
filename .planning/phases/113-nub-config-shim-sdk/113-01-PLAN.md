Mirrors `packages/nubs/identity/src/shim.ts` (correlation-id Map pattern) and `packages/nubs/notify/src/shim.ts` (fan-out Set pattern), fused into a single file that handles the config domain's three distinct behaviors:

1. **Request/response correlation** ‚Äî `config.get` and `config.registerSchema` track pending promises in per-kind Maps keyed on UUID `id`; resolved by matching `config.values` (with id) and `config.registerSchema.result` messages.

### 5. Shell-message router (exported)

```ts
/**
 * Handle config.* messages from the shell. Called by the central shim dispatcher.
 */
export function handleConfigMessage(msg: { type: string; [key: string]: unknown }): void {
  const type = msg.type;
  if (type === 'config.registerSchema.result') {
    handleRegisterSchemaResult(msg as unknown as ConfigRegisterSchemaResultMessage);
  } else if (type === 'config.values') {
    handleValues(msg as unknown as ConfigValuesMessage);
  } else if (type === 'config.schemaError') {
    handleSchemaError(msg as unknown as ConfigSchemaErrorMessage);
  }
}
```

### 6. Internal handlers (below the router)

**handleRegisterSchemaResult**: look up pending registration by `msg.id`; on `ok:true` set `currentSchema = pending.schema` and resolve; on `ok:false` reject with `new Error(msg.code + ': ' + (msg.error ?? 'schema rejected'))`. Always delete from map.

**handleValues**: update `lastValues = msg.values` unconditionally. If `msg.id` is present, this is a correlated response to config.get ‚Äî look up `pendingGets.get(msg.id)`, resolve with values, delete from map. If `msg.id` is absent, this is a subscription push ‚Äî for-of iterate `subscribers` and invoke each callback with `msg.values`.

**handleSchemaError**: for-of iterate `schemaErrorHandlers` and invoke with `{ code: msg.code, error: msg.error }`.

### 7. Meta-tag schema read helper

```ts
/**
 * Read the manifest-declared schema from <meta name="napplet-config-schema"> if present.
 * Silent on any failure (missing tag, missing content, invalid JSON) -- schema remains null.
 */
function readManifestSchema(): NappletConfigSchema | null {
  if (typeof document === 'undefined') return null;
  const el = document.querySelector(`meta[name="${SCHEMA_META_NAME}"]`);
  if (!el) return null;
  const raw = el.getAttribute('content');
  if (!raw) return null;
  try {
    return JSON.parse(raw) as NappletConfigSchema;
  } catch {
    return null;
  }
}
```

### 8. API implementations (match the NappletConfig interface from the spec exactly)

**get()**: generate `id = crypto.randomUUID()`; create Promise; register resolver in `pendingGets`; postMessage `{ type: 'config.get', id } satisfies ConfigGetMessage`; setTimeout-reject after `REQUEST_TIMEOUT_MS` (delete on timeout).

**subscribe(cb)**: add cb to `subscribers` Set. If `subscribers.size === 1` (0‚Üí1 transition), postMessage `{ type: 'config.subscribe' } satisfies ConfigSubscribeMessage`. If `lastValues` is already cached (late subscriber after a push), schedule `queueMicrotask(() => { if (subscribers.has(cb) && lastValues !== null) cb(lastValues); })` so the subscriber still receives an initial snapshot without waiting for the shell's next push. Return `{ close(): void }` that deletes cb from `subscribers`; on 1‚Üí0 transition postMessage `{ type: 'config.unsubscribe' } satisfies ConfigUnsubscribeMessage`. Return type is `Subscription` from `@napplet/core`.

**registerSchema(schema, version?)**: `id = crypto.randomUUID()`; create Promise<void>; store `{ resolve, reject, schema }` in `pendingRegistrations`; postMessage `{ type: 'config.registerSchema', id, schema, version } satisfies ConfigRegisterSchemaMessage` (omit version key if `undefined`); setTimeout-reject after `REQUEST_TIMEOUT_MS`.

**openSettings(options?)**: postMessage `{ type: 'config.openSettings', section: options?.section } satisfies ConfigOpenSettingsMessage` (omit section key if undefined). Fire-and-forget. Returns `void`.

**onSchemaError(cb)**: add cb to `schemaErrorHandlers`. Return a teardown function `() => { schemaErrorHandlers.delete(cb); }`. Per the merged spec, `onSchemaError` returns a plain teardown `() => void` (not a `Subscription`) ‚Äî match the merged API surface exactly.

**getSchema()**: not exported as a named function ‚Äî the `schema` accessor is a getter property on the mounted object (see mount step below).

### 9. installConfigShim (exported)

```ts
/**
 * Install the config shim: read manifest-declared schema, mount window.napplet.config.
 * Idempotent -- double-install is a no-op.
 *
 * @returns cleanup function that clears all state and removes the window mount
 */
export function installConfigShim(): () => void {
  if (installed) {
    return () => { /* already installed */ };
  }

  // 1. Read manifest-declared schema synchronously.
  currentSchema = readManifestSchema();

  // 2. Mount window.napplet.config. Use Object.defineProperty for the readonly `schema`
  //    accessor so authors reading window.napplet.config.schema at any time get the
  //    current cached value (updated by successful registerSchema responses).
  const napplet = (window as any).napplet ?? ((window as any).napplet = {});
  const api: Record<string, unknown> = {
    registerSchema,
    get,
    subscribe,
    openSettings,
    onSchemaError,
  };
  Object.defineProperty(api, 'schema', {
    get: () => currentSchema,
    enumerable: true,
    configurable: false,
  });
  napplet.config = api;

  installed = true;

  return () => {
    subscribers.clear();
    schemaErrorHandlers.clear();
    pendingGets.clear();
    pendingRegistrations.clear();
    lastValues = null;
    currentSchema = null;
    if (napplet.config === api) delete napplet.config;
    installed = false;
  };
}
```
