# Plan 42-02: Implement SDK wrapper exports and type re-exports

<objective>
Implement `packages/sdk/src/index.ts` with four namespaced export objects (`relay`, `ipc`, `services`, `storage`) that delegate to `window.napplet.*` at call time with runtime guards, re-export all public protocol types from `@napplet/core`, and include a global Window type augmentation for `window.napplet` autocompletion.
</objective>

## Tasks

<task id="42-02-01">
<title>Create SDK entry point with runtime guard and relay wrapper</title>
<read_first>
- packages/shim/src/relay-shim.ts
- packages/core/src/types.ts
- packages/core/src/index.ts
- .planning/phases/42-sdk-package/42-CONTEXT.md
- .planning/phases/41-shim-restructure/41-CONTEXT.md
</read_first>
<action>
Create `packages/sdk/src/index.ts` with the full SDK implementation.

The file structure is:

1. **Module header comment** explaining SDK purpose
2. **Import `NappletGlobal` from `@napplet/core`** (added by Phase 41)
3. **Runtime guard function:**
   ```typescript
   function requireNapplet(): NappletGlobal {
     const w = window as Window & { napplet?: NappletGlobal };
     if (!w.napplet) {
       throw new Error('window.napplet not installed -- import @napplet/shim first');
     }
     return w.napplet;
   }
   ```
4. **relay export object** — `subscribe`, `publish`, `query` methods that delegate to `requireNapplet().relay.*`:
   ```typescript
   export const relay = {
     subscribe(
       filters: NostrFilter | NostrFilter[],
       onEvent: (event: NostrEvent) => void,
       onEose: () => void,
       options?: { relay?: string; group?: string },
     ): Subscription {
       return requireNapplet().relay.subscribe(filters, onEvent, onEose, options);
     },
     publish(
       template: EventTemplate,
       options?: { relay?: boolean },
     ): Promise<NostrEvent> {
       return requireNapplet().relay.publish(template, options);
     },
     query(filters: NostrFilter | NostrFilter[]): Promise<NostrEvent[]> {
       return requireNapplet().relay.query(filters);
     },
   };
   ```
5. **ipc export object** — `emit`, `on` methods delegating to `requireNapplet().ipc.*`:
   ```typescript
   export const ipc = {
     emit(topic: string, extraTags?: string[][], content?: string): void {
       requireNapplet().ipc.emit(topic, extraTags, content);
     },
     on(
       topic: string,
       callback: (payload: unknown, event: NostrEvent) => void,
     ): { close(): void } {
       return requireNapplet().ipc.on(topic, callback);
     },
   };
   ```
6. **services export object** — `list`, `has` methods delegating to `requireNapplet().services.*`:
   ```typescript
   export const services = {
     list(): Promise<ServiceInfo[]> {
       return requireNapplet().services.list();
     },
     has(name: string, version?: string): Promise<boolean> {
       return requireNapplet().services.has(name, version);
     },
   };
   ```
7. **storage export object** — `getItem`, `setItem`, `removeItem`, `keys` methods delegating to `requireNapplet().storage.*`:
   ```typescript
   export const storage = {
     getItem(key: string): Promise<string | null> {
       return requireNapplet().storage.getItem(key);
     },
     setItem(key: string, value: string): Promise<void> {
       return requireNapplet().storage.setItem(key, value);
     },
     removeItem(key: string): Promise<void> {
       return requireNapplet().storage.removeItem(key);
     },
     keys(): Promise<string[]> {
       return requireNapplet().storage.keys();
     },
   };
   ```
8. **Type re-exports from `@napplet/core`:**
   ```typescript
   export type { NostrEvent, NostrFilter } from '@napplet/core';
   export type { ServiceInfo } from '@napplet/core';
   export type { Subscription, EventTemplate } from '@napplet/core';
   ```
   Note: `ServiceInfo`, `Subscription`, and `EventTemplate` must exist in `@napplet/core` after Phase 41. If Phase 41 named the service type `ServiceDescriptor`, re-export it aliased as `ServiceInfo`:
   ```typescript
   export type { ServiceDescriptor as ServiceInfo } from '@napplet/core';
   ```
9. **Global Window type augmentation:**
   ```typescript
   declare global {
     interface Window {
       napplet: NappletGlobal;
     }
   }
   ```

The complete file should be approximately 100-120 lines. Each method is explicit (no Proxy magic), follows camelCase naming, and includes JSDoc with `@param`, `@returns`, and `@example` tags on each public export object.
</action>
