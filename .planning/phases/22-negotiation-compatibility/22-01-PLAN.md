<interfaces>
The Nip5aManifestOptions interface gets a new optional field:

```typescript
export interface Nip5aManifestOptions {
  /** Napp type/dtag (e.g., 'feed', 'chat') */
  nappType: string;
  /** Service dependencies this napplet requires (e.g., ['audio', 'notifications']) */
  requires?: string[];
}
```

The vite plugin:
1. In `transformIndexHtml()`: if `requires` has entries, inject a `<meta name="napplet-requires" content="audio,notifications">` tag (comma-separated)
2. In `closeBundle()`: inject `["requires", "service-name"]` tags into the manifest event's tags array, one tag per service name

Tag format follows CONTEXT D-04: `["requires", "service-name"]` — name only, no version.
</interfaces>

    Change the return array from:
    ```typescript
    transformIndexHtml(): IndexHtmlTransformResult {
      // Inject meta tag (empty in dev, populated in closeBundle for build)
      return [
        {
          tag: 'meta',
          attrs: {
            name: 'napplet-aggregate-hash',

        },
      ];
    },
    ```

    To:
    ```typescript
    transformIndexHtml(): IndexHtmlTransformResult {
      // Inject meta tag (empty in dev, populated in closeBundle for build)
      const tags: IndexHtmlTransformResult = [
        {
          tag: 'meta',
          attrs: {
            name: 'napplet-aggregate-hash',
            content: '',
          },
          injectTo: 'head' as const,
        },
      ];

      // Inject requires meta tag if service dependencies are declared
      if (options.requires && options.requires.length > 0) {
        tags.push({
          tag: 'meta',
          attrs: {
            name: 'napplet-requires',
            content: options.requires.join(','),
          },
          injectTo: 'head' as const,
        });
      }

      return tags;
    },
    ```
