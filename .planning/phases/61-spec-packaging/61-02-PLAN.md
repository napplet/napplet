# Plan 61-02: Finalize NIP-5D Format and Add Implementations Section

<objective>
Ensure the NIP-5D draft file (created in Phase 58, extended in Phase 59) conforms to nostr-protocol/nips markdown conventions, add a References section and an Implementations section listing the two reference implementations, and merge any separate pipe protocol document into the main NIP file. The result is a single self-contained NIP markdown file ready for PR submission.
</objective>

<read_first>

- NIP-5D-PIPES.md (if it exists — the pipe protocol section from Phase 59)

- .planning/phases/58-core-protocol-nip/58-CONTEXT.md (D-06: NIP number is 5D)
</read_first>
<action>
Locate the NIP draft file. Phase 58 should have created it (likely as `NIP-5D.md` or `5D.md` in the repo root). If it has a different name, use whatever file Phase 58 produced.

Audit the file against these nostr-protocol/nips conventions:

1. **Setext heading format** — First two headings MUST use setext (underline) style:
   ```markdown
   NIP-5D
   ======
   
   Nostr Web Applets
   -----------------
   ```
   The first heading is the NIP identifier. The second is the title. Both use setext.

2. **Status badges** — Must have `draft` `optional` (backtick-wrapped, space-separated) immediately after the subtitle, with a blank line before the body. No YAML frontmatter, no author fields, no date fields.

3. **Body headings** — All section headings in the body use ATX style (`###` level, NOT `##`). This matches NIP-5A's convention where body sections use `### Section Name`.

4. **RFC 2119 keywords** — MUST, SHOULD, MAY in uppercase when used normatively.

5. **NIP references** — Use relative links: `[NIP-01](01.md)`, `[NIP-5A](5A.md)`, `[NIP-42](42.md)`. Not full GitHub URLs.

6. **Code blocks** — Use `jsonc` or `json` language hint for JSON examples.

7. **Line count** — Should be under 500 lines total.

If any of these conventions are not met, note what needs to be fixed. Apply fixes in subsequent tasks.
</action>
<acceptance_criteria>
- The NIP draft file is located and its path is identified
- An audit checklist is mentally tracked for what needs fixing
- If the file already fully conforms, no changes needed in this task
</acceptance_criteria>
</task>

## Verification

```bash
# Locate NIP file
NIP_FILE=$(ls NIP-5D.md 5D.md 2>/dev/null | head -1)
test -n "$NIP_FILE" && echo "PASS: NIP file found: $NIP_FILE" || echo "FAIL: No NIP file"

# Header format
head -1 "$NIP_FILE" | grep -q "^NIP-5D$" && echo "PASS: correct NIP identifier"
head -2 "$NIP_FILE" | tail -1 | grep -q "^======$" && echo "PASS: setext underline"

# Status badges
grep -q '`draft` `optional`' "$NIP_FILE" && echo "PASS: status badges"

# No full GitHub URLs for NIP references
stale_urls=$(grep -c "github.com/nostr-protocol/nips/blob" "$NIP_FILE" 2>/dev/null || echo 0)
echo "Full NIP URLs remaining: $stale_urls (should be 0)"

# Implementations section
grep -q "### Implementations" "$NIP_FILE" && echo "PASS: Implementations section"
grep -q "napplet/shim" "$NIP_FILE" && echo "PASS: SDK listed"
grep -q "hyprgate" "$NIP_FILE" && echo "PASS: hyprgate listed"

# Line count
lines=$(wc -l < "$NIP_FILE")
echo "NIP file: $lines lines"
test "$lines" -lt 500 && echo "PASS: under 500 lines" || echo "WARN: over 500 lines"

# No separate pipe file
test ! -f NIP-5D-PIPES.md && echo "PASS: no separate pipe file" || echo "WARN: separate pipe file still exists"

# Pipe section present
grep -q "PIPE_OPEN\|pipes" "$NIP_FILE" && echo "PASS: pipe section present"

# Body heading level
bad_headings=$(grep -c "^## " "$NIP_FILE" 2>/dev/null || echo 0)
echo "## headings in body: $bad_headings (should be 0)"
```

- [ ] All body sections use `### ` ATX headings (not `## `)
- [ ] NIP cross-references use relative links (`[NIP-01](01.md)`) not full GitHub URLs
- [ ] Implementations section lists @napplet/shim + @napplet/shell (SDK) and hyprgate (reference shell) with GitHub links
