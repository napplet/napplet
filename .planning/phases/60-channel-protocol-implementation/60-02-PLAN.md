Create `packages/runtime/src/pipe-handler.ts` with the following structure:

```typescript
/**
 * pipe-handler.ts — Pipe protocol handler for inter-napplet communication.
 *
 * Manages pipe lifecycle: PIPE_OPEN -> PIPE_ACK -> PIPE -> PIPE_CLOSE
 * plus PIPE_BROADCAST fan-out. All pipes are shell-mediated — no direct
 * iframe-to-iframe communication.
 */

import type { SendToNapplet } from './types.js';
import type { SessionRegistry } from './session-registry.js';
import {
  VERB_PIPE_ACK,
  VERB_PIPE,
  VERB_PIPE_CLOSE,
} from '@napplet/core';
```

**Data structures:**

```typescript
/** A single pipe connection between two napplet endpoints. */
export interface PipeEntry {
  /** Shell-assigned opaque pipe identifier. */
  id: string;
  /** WindowId of the napplet that initiated PIPE_OPEN. */
  endpointA: string;
  /** WindowId of the target napplet. */
  endpointB: string;
  /** Optional named pipe for multiplexing. */
  name?: string;
}

/** Pipe registry — all active pipes. */
const pipes = new Map<string, PipeEntry>();

/** Reverse index: windowId -> Set<pipeId> for fast cleanup. */
const windowPipes = new Map<string, Set<string>>();
```

**Functions to implement:**

1. `createPipeHandler(deps)` — factory function (follows createRuntime pattern). Takes:
   - `sendToNapplet: SendToNapplet`
   - `sessionRegistry: SessionRegistry`
   - `enforce: (pubkey: string, cap: string, msg: unknown[]) => { allowed: boolean; capability: string }`
   - `generateId: () => string` (from hooks.crypto.randomUUID)

   Returns an object with methods:

2. `handlePipeOpen(windowId: string, msg: unknown[]): void`
   - Extract options from msg[1]: `{ targetType: string, targetKey?: string, name?: string }`
   - Validate sender is authenticated: `sessionRegistry.getPubkey(windowId)`. If null, send `["PIPE_CLOSE", null, "auth-required"]`
   - Enforce `pipe:connect` capability. If denied, send `["PIPE_CLOSE", null, "acl-denied"]`
   - Resolve target:
     - If `targetKey` provided: `sessionRegistry.getEntry(targetKey)` and verify it matches targetType
     - If only `targetType`: iterate all session entries to find one with matching dTag, excluding the sender's windowId
     - If no match found: send `["PIPE_CLOSE", null, "target-not-found"]`
   - Generate pipeId: `'p-' + generateId().slice(0, 8)`
   - Create PipeEntry and add to pipes Map
   - Add pipeId to both windows' windowPipes Sets
   - Get sender's session entry for peer info
   - Send PIPE_ACK to endpoint A (sender): `["PIPE_ACK", { pipeId, peer: { pubkey: targetEntry.pubkey, dTag: targetEntry.dTag } }]`
   - Send PIPE_ACK to endpoint B (target): `["PIPE_ACK", { pipeId, peer: { pubkey: senderEntry.pubkey, dTag: senderEntry.dTag } }]`

3. `handlePipe(windowId: string, msg: unknown[]): void`
   - Extract pipeId from msg[1], payload from msg[2]
   - Look up pipe entry by pipeId. If not found, silently return (invalid pipe)
   - Determine peer: if windowId === endpointA, peer is endpointB; else peer is endpointA
   - Verify sender is actually one of the endpoints. If not, silently return
   - Forward: `sendToNapplet(peerWindowId, ["PIPE", pipeId, payload])`

4. `handlePipeClose(windowId: string, msg: unknown[]): void`
   - Extract pipeId from msg[1]
   - Look up pipe entry. If not found, silently return
   - Verify sender is an endpoint
   - Determine peer windowId
   - Remove pipe from registry and both windowPipes Sets
   - Send PIPE_CLOSE to peer: `sendToNapplet(peerWindowId, ["PIPE_CLOSE", pipeId, ""])`

5. `handlePipeBroadcast(windowId: string, msg: unknown[]): void`
   - Extract payload from msg[1]
   - Check sender is authenticated (getPubkey). If not, silently return
   - Get all pipeIds for this window from windowPipes
   - For each pipe, determine the peer windowId (the other endpoint)
   - Send `["PIPE", pipeId, payload]` to each peer
   - Sender is excluded (they sent the broadcast)

6. `cleanupWindowPipes(windowId: string): void`
   - Get all pipeIds for this window from windowPipes
   - For each pipe:
     - Determine peer
     - Send `["PIPE_CLOSE", pipeId, "peer-disconnected"]` to peer
     - Remove from pipes Map
     - Remove from peer's windowPipes Set
   - Delete window from windowPipes Map

7. `getPipeCount(): number` — returns pipes.size (for testing)

**2. Update `packages/runtime/src/enforce.ts`:**

In the `resolveCapabilities()` function's switch statement, add cases for all pipe verbs BEFORE the `default` case. This prevents them from falling through to the default `relay:write` requirement:

```typescript
    // Pipe verbs — auth/ACL checked directly by pipe handler, not by resolveCapabilities.
    // PIPE_OPEN requires pipe:connect (enforced in pipe-handler.ts).
    // PIPE, PIPE_CLOSE, PIPE_BROADCAST require no additional capability — pipe:connect
    // is checked once at open time. Data flow on established pipes is unrestricted.
    case 'PIPE_OPEN':
      return { senderCap: 'pipe:connect', recipientCap: null };
    case 'PIPE':
    case 'PIPE_CLOSE':
    case 'PIPE_BROADCAST':
      return { senderCap: null, recipientCap: null };
```

Add the `VERB_PIPE_OPEN`, `VERB_PIPE`, `VERB_PIPE_CLOSE`, `VERB_PIPE_BROADCAST` imports from `@napplet/core` at the top of enforce.ts. Alternatively, use the string literals directly in the switch cases (they are `'PIPE_OPEN'`, `'PIPE'`, `'PIPE_CLOSE'`, `'PIPE_BROADCAST'`).

**Important:** The pipe handler calls `enforce()` directly in `handlePipeOpen()` for the actual ACL check. The `resolveCapabilities()` entry here is for correctness if dispatchVerb ever calls resolveCapabilities on pipe verbs, and to prevent the default case from incorrectly requiring relay:write.
