**Authentication helper for tests:**

The tests need to authenticate windows with specific pubkeys and dTags. Since Phase 46 uses REGISTER -> IDENTITY -> AUTH flow, the helper needs to:
1. Send REGISTER with dTag
2. Find the IDENTITY response to get the delegated pubkey
3. Send AUTH with the delegated pubkey (mock verifyEvent returns true)
4. Wait for OK

The mock adapter's crypto.verifyEvent always returns true, so signature verification is bypassed.

Each test that needs pipes should:
1. Authenticate two (or three) windows
2. Send PIPE_OPEN from one
3. Extract pipeId from PIPE_ACK in `mock.sent`
4. Proceed with PIPE/PIPE_CLOSE/PIPE_BROADCAST operations

**Important test infrastructure notes:**
- Use `await new Promise(resolve => setTimeout(resolve, 10))` after async operations (AUTH is async due to verifyEvent)
- Filter `mock.sent` by windowId and verb to find specific messages
- Reset mock between test cases via `beforeEach`
