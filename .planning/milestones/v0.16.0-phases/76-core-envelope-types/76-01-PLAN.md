    - registerNub('relay', handler) stores the handler keyed by 'relay'
    - registerNub('relay', handler2) when 'relay' already registered throws Error('NUB domain "relay" is already registered')
    - dispatch({ type: 'relay.subscribe' }) calls the relay handler with the message, returns true
    - dispatch({ type: 'signer.sign' }) when no signer handler registered returns false
    - dispatch({ type: 'malformed' }) with no dot in type returns false (no domain match)
    - dispatch({ type: '.action' }) with empty domain prefix returns false
    - getRegisteredDomains() returns array of registered domain strings
    - createDispatch() returns a fresh { registerNub, dispatch, getRegisteredDomains } instance (factory pattern for testability and multiple registries)
  </behavior>
  <action>
Create packages/core/src/dispatch.ts with:

1. **NubHandler type**: `(message: NappletMessage) => void` -- the callback a NUB module provides to handle messages in its domain.

2. **createDispatch() factory** that returns `{ registerNub, dispatch, getRegisteredDomains }`:
   - Internal state: `Map<string, NubHandler>` keyed by domain string
   - `registerNub(domain: string, handler: NubHandler): void` -- stores handler, throws if domain already registered
   - `dispatch(message: NappletMessage): boolean` -- extracts domain from `message.type` by splitting on first `.`, looks up handler, calls it, returns true. Returns false if no handler found or invalid type format.
   - `getRegisteredDomains(): string[]` -- returns Array.from of the map keys

3. **Module-level singleton** for convenience: export a default registry instance's methods as named exports (`registerNub`, `dispatch`, `getRegisteredDomains`) alongside the `createDispatch` factory.

Follow project conventions:
- camelCase function names, PascalCase types
- JSDoc with @param, @returns, @example on all exports
- Section dividers using `// --- Section Name ---` format
- 2-space indentation, semicolons
