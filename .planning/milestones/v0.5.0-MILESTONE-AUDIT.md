## Integration Issues

### HIGH — Functional Correctness Risk

**[README-03, SKILL-02] originRegistry.register() argument order inverted**
- Location: `packages/shell/README.md` line 92
- Shown: `originRegistry.register(windowId, iframe.contentWindow!)`
- Correct: `register(win: Window, windowId: string)` — Window is the first argument
- The SKILL-02 file has the correct argument order; shell README has it wrong.
- A developer using the shell README as reference will write broken code.

### MEDIUM — Documentation Completeness

**[README-03, SKILL-02] ShellBridge.runtime property missing from shell README**
- `packages/shell/README.md` method table (lines 117-126) does not document `readonly runtime: Runtime`
- This property is the only path to `registerService()` dynamically
- SKILL-02 uses `bridge.runtime.registerService()` but the shell README doesn't document it

**[SPEC-01] Section 3.8 normative kind table missing kind 29010**
- `SPEC.md` Section 3.8 lists ephemeral bus kinds 29000-29007 only
- Kind 29010 (`SERVICE_DISCOVERY`) is absent from the normative table
- It appears in Section 11 (fully documented) and Appendix 15, but the authoritative Section 3.8 table is incomplete

### LOW — Coherence, No Runtime Break

**[README-02, SKILL-01] nappState vs nappStorage naming**
- Primary export: `nappState` (shim README, root README)
- Alias: `nappStorage` (SKILL-01 frontmatter, all Step 6 examples)
- Both work at runtime (`export const nappStorage = nappState` in state-shim.ts)
- Documentation does not explain the alias relationship
