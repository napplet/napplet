# Plan 42-01: Scaffold @napplet/sdk workspace package

<objective>
Create the `packages/sdk/` directory with `package.json`, `tsconfig.json`, and `tsup.config.ts` following existing workspace conventions. The package depends only on `@napplet/core` (no shim dependency) and produces ESM-only output with TypeScript declarations.
</objective>

## Tasks

<task id="42-01-01">
<title>Create packages/sdk/package.json</title>
<read_first>
- packages/shim/package.json
- packages/core/package.json
</read_first>
<action>
Create `packages/sdk/package.json` with the following exact content:

```json
{
  "name": "@napplet/sdk",
  "version": "0.1.0",
  "description": "Typed named exports wrapping window.napplet for bundler-consuming napplet developers",
  "type": "module",
  "main": "./dist/index.js",
  "module": "./dist/index.js",
  "types": "./dist/index.d.ts",
  "exports": {
    ".": {
      "types": "./dist/index.d.ts",
      "import": "./dist/index.js"
    }
  },
  "files": ["dist"],
  "sideEffects": false,
  "publishConfig": {
    "access": "public"
  },
  "dependencies": {
    "@napplet/core": "workspace:*"
  },
  "devDependencies": {
    "tsup": "^8.5.0",
    "typescript": "^5.9.3"
  },
  "scripts": {
    "build": "tsup",
    "type-check": "tsc --noEmit"
  },
  "license": "MIT",
  "repository": {
    "type": "git",
    "url": "git+https://github.com/sandwichfarm/napplet.git",
    "directory": "packages/sdk"
  },
  "keywords": ["nostr", "napplet", "sdk", "iframe", "nip-01"]
}
```

Key differences from shim:
- `"sideEffects": false` (SDK has no side effects; shim is `true`)
- No `nostr-tools` peer dependency (SDK delegates to window.napplet at runtime)
- No `@napplet/shim` dependency (PKG-03)
- Only `@napplet/core` as runtime dependency
</action>
<acceptance_criteria>
- `packages/sdk/package.json` exists
- `grep -c '"name": "@napplet/sdk"' packages/sdk/package.json` returns 1
- `grep -c '"sideEffects": false' packages/sdk/package.json` returns 1
- `grep -c '@napplet/shim' packages/sdk/package.json` returns 0
- `grep -c '"@napplet/core": "workspace:\*"' packages/sdk/package.json` returns 1
- `grep -c 'nostr-tools' packages/sdk/package.json` returns 0
</acceptance_criteria>
</task>

<task id="42-01-02">
<title>Create packages/sdk/tsconfig.json</title>
<read_first>
- packages/shim/tsconfig.json
- tsconfig.json
</read_first>
<action>
Create `packages/sdk/tsconfig.json` with the following exact content:

```json
{
  "extends": "../../tsconfig.json",
  "compilerOptions": {
    "outDir": "dist",
    "rootDir": "src",
    "lib": ["ES2022", "DOM", "DOM.Iterable"]
  },
  "include": ["src"]
}
```

The `DOM` lib is required because SDK references `window` in its runtime guard function.
</action>
<acceptance_criteria>
- `packages/sdk/tsconfig.json` exists
- `grep -c '"DOM"' packages/sdk/tsconfig.json` returns at least 1
- `grep -c '"extends": "../../tsconfig.json"' packages/sdk/tsconfig.json` returns 1
</acceptance_criteria>
</task>

<task id="42-01-03">

Identical to shim's tsup config — same ESM-only, dts, sourcemap, clean pattern.
</action>

</task>
