# Plan 43-03: Build and Test Verification

## Objective

Verify that all migrated demo napplets and test fixtures build, type-check, and pass the full e2e test suite. This plan produces no file changes — it runs the verification pipeline and fixes any issues surfaced by the build or tests.

## Tasks

<task id="43-03-01">
<title>Run pnpm install to update workspace dependencies</title>

<read_first>
- apps/demo/napplets/chat/package.json
- apps/demo/napplets/bot/package.json
</read_first>

<action>
Run `pnpm install` from the repo root to resolve the new `@napplet/sdk` workspace dependency added to the demo napplet `package.json` files. This updates the lockfile and creates the necessary symlinks in `node_modules/`.

```bash
pnpm install
```

If pnpm install fails because `@napplet/sdk` package directory doesn't exist yet (Phase 42 not yet executed), this is a blocker — Phase 43 depends on Phase 42 completing first. Report the blocker and stop.
</action>

<acceptance_criteria>
- `pnpm install` exits with code 0
- `ls apps/demo/napplets/chat/node_modules/@napplet/sdk` exists (symlink or directory)
- `ls apps/demo/napplets/bot/node_modules/@napplet/sdk` exists (symlink or directory)
</acceptance_criteria>
</task>

<task id="43-03-02">
<title>Run type-check across all packages</title>

<read_first>
- tsconfig.json (root)
</read_first>

<action>
Run the TypeScript type-checker to verify all migrated files compile without errors:

```bash
pnpm type-check
```

If type errors occur in the migrated files:
- **Import resolution errors** (`Cannot find module '@napplet/sdk'`): Verify Phase 42 created the package with proper exports
- **Type errors on `relay.publish()` etc.**: Verify SDK exports match the expected signatures from Phase 42 CONTEXT
- **`window.napplet` type errors in publish-napplet**: May need to add `/// <reference types="@napplet/shim" />` or verify the Window augmentation from Phase 41 is working

Fix any type errors in the migrated files (not in SDK/shim source — those are prior phase scope).
</action>

<acceptance_criteria>
- `pnpm type-check` exits with code 0
- No TypeScript errors in `apps/demo/napplets/chat/src/main.ts`
- No TypeScript errors in `apps/demo/napplets/bot/src/main.ts`
- No TypeScript errors in `tests/fixtures/napplets/publish-napplet/src/main.ts`
</acceptance_criteria>
</task>

<task id="43-03-03">
<title>Run full build</title>

<read_first>
- turbo.json
</read_first>

<action>
Run the full build pipeline to verify all packages and demo napplets compile:

```bash
pnpm build
```

This runs turborepo which builds all packages in dependency order: core → runtime → acl → services → shim → sdk → shell → vite-plugin → demo napplets → test fixtures.

If build errors occur, diagnose and fix. Common issues:
- Vite alias resolution failures in demo napplet builds
- Missing SDK package in vite config resolve paths
</action>

<acceptance_criteria>
- `pnpm build` exits with code 0
- `ls apps/demo/napplets/chat/dist/` contains built output
- `ls apps/demo/napplets/bot/dist/` contains built output
</acceptance_criteria>
</task>

<task id="43-03-04">
<title>Run e2e test suite</title>

<read_first>

- tests/e2e/demo-audit-correctness.spec.ts
</read_first>

<action>
Run the full Playwright e2e test suite to verify the migrated code works in a real browser:

```bash
pnpm test:e2e
```

All 18 e2e spec files must pass. Key tests to watch:
- `auth.spec.ts` — Loads `auth-napplet` (no migration, but verifies shim side-effect import still works)
- `demo-audit-correctness.spec.ts` — Tests demo chat/bot functionality through the full shell pipeline
- `demo-notification-service.spec.ts` — Tests notification flow through `ipc.emit('notifications:create', ...)`
- `signer-delegation.spec.ts` — Uses `publish-napplet` fixture

If e2e tests fail:
- Check if the failure is in a test that exercises migrated code (demo or publish-napplet)
- Verify the window.napplet global is populated before the fixture code runs
- Check browser console for runtime errors (SDK guard throwing, missing window.napplet, etc.)

Fix any issues in the migrated files or configs (not in test harness or shell — those are out of scope).
</action>

<acceptance_criteria>
- `pnpm test:e2e` exits with code 0
- All 18 e2e spec files report passed
- No test regressions compared to pre-migration state
</acceptance_criteria>
</task>

<task id="43-03-05">
<title>Final grep audit — zero old API references</title>

<read_first>
- apps/demo/napplets/chat/src/main.ts
- apps/demo/napplets/bot/src/main.ts
- tests/fixtures/napplets/publish-napplet/src/main.ts
</read_first>

<action>
Run a comprehensive grep audit to verify no old shim named exports remain in demo or test code:

```bash
# 1. No named imports from @napplet/shim in demos
grep -r "from '@napplet/shim'" apps/demo/ --include="*.ts" | grep -v node_modules

# 2. No named imports from @napplet/shim in test fixtures
grep -r "from '@napplet/shim'" tests/fixtures/ --include="*.ts" | grep -v node_modules

# 3. No bare publish/subscribe/emit/on/nappState calls (must be prefixed)
grep -rn "\bpublish(" apps/demo/ --include="*.ts" | grep -v node_modules | grep -v "relay\.publish"
grep -rn "\bsubscribe(" apps/demo/ --include="*.ts" | grep -v node_modules | grep -v "relay\.subscribe"
grep -rn "\bemit(" apps/demo/ --include="*.ts" | grep -v node_modules | grep -v "ipc\.emit"
grep -rn "\bon(" apps/demo/ --include="*.ts" | grep -v node_modules | grep -v "ipc\.on" | grep -v addEventListener
grep -rn "\bnappState\b" apps/demo/ --include="*.ts" | grep -v node_modules
grep -rn "\bnappStorage\b" apps/demo/ --include="*.ts" | grep -v node_modules
grep -rn "\bnappletState\b" apps/demo/ --include="*.ts" | grep -v node_modules

# 4. No bare publish call in test fixtures
grep -rn "\bpublish(" tests/fixtures/ --include="*.ts" | grep -v node_modules | grep -v "relay\.publish"
```

All of the above grep commands should return zero results. If any results are found, go back and fix the missed call sites.
</action>

<acceptance_criteria>
- `grep -rc "from '@napplet/shim'" apps/demo/ --include="*.ts" | grep -v node_modules | grep -v ":0$"` returns no output (all files show 0 matches)
- `grep -rc "from '@napplet/shim'" tests/fixtures/ --include="*.ts" | grep -v node_modules | grep -v ":0$"` returns no output
- `grep -rn "nappState\|nappStorage\|nappletState" apps/demo/ tests/fixtures/ --include="*.ts" | grep -v node_modules` returns no output
- `grep -rn "discoverServices\|hasService\b\|hasServiceVersion" apps/demo/ tests/fixtures/ --include="*.ts" | grep -v node_modules` returns no output
</acceptance_criteria>
</task>
