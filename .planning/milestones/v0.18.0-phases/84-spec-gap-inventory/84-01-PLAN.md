# Plan 84-01: Create SPEC-GAPS.md Inventory Document

<objective>
Create the complete spec gap inventory document at `.planning/SPEC-GAPS.md` that catalogs every piece of code not covered by NIP-5D or any NUB spec, with location, purpose, evidence, and recommendation category.
</objective>

<context>
This phase is pure documentation — no code changes. The output is a single `.planning/SPEC-GAPS.md` file consumed by Phase 86 (Decision Gate) where the spec author makes drop-or-amend decisions on each gap.

Four recommendation categories (from CONTEXT.md D-04):
- `future-nub` — Functionality that will become NUB messages
- `unknown` — No clear home yet, needs decision
- `superseded` — Replaced by existing NUB messages or removed protocol features
- `shell-only` — Shell implementation concern, not an SDK/spec concern
</context>
