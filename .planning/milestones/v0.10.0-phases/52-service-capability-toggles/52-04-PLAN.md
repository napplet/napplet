# Plan 52-04: Integration Wiring and Cross-View Sync

<objective>
Wire all toggle UI components together in the demo's main entry point. Connect the topology service toggle icons to shell-host.toggleService(), sync topology visuals when services are toggled from the modal, and sync modal state when capabilities are toggled from inline ACL buttons. Ensure all toggle paths produce consistent visual state across all views.
</objective>

## Tasks

<task id="52-04-01">
<title>Wire topology service toggles in main.ts</title>
<read_first>
- apps/demo/src/main.ts
- apps/demo/src/topology.ts (wireServiceToggles, updateServiceNodeVisual)
- apps/demo/src/shell-host.ts (toggleService, isServiceEnabled)
</read_first>

<action>
In `apps/demo/src/main.ts`, locate where `initTopologyEdges()` is called after topology rendering. After that call, add a call to `wireServiceToggles()` with a callback that delegates to `toggleService()`.

First, add the imports. Find the existing topology imports and add `wireServiceToggles` and `updateServiceNodeVisual`:

In the import from `./topology.js`, add `wireServiceToggles` to the imported names.

Then, after the `initTopologyEdges()` call, add:

```typescript
  // Wire service toggle icons on topology nodes
  wireServiceToggles((name, enabled) => {
    toggleService(name, enabled);
  });
```

Also add `toggleService` to the imports from `./shell-host.js` if not already present.

Specifically, find the location in main.ts where topology edges are initialized (likely after `renderDemoTopology` output is inserted into the DOM). The wireServiceToggles call must come AFTER the topology HTML is in the DOM, because it queries for `.service-toggle-icon` elements.
</action>

- `pnpm build` succeeds
- `pnpm type-check` succeeds
</acceptance_criteria>
</task>
