# Plan 49-03: Wire Config Into Demo Modules

<objective>
Replace hardcoded module-level constants in demo source files with live reads from the demoConfig object, so that editing a constant in the panel takes immediate effect on subsequent operations (flash timing, toast display, signer request window, diagram layout).
</objective>

## Tasks

<task id="49-03-01">
<title>Wire FLASH_DURATION in flow-animator.ts</title>
<read_first>
- apps/demo/src/flow-animator.ts
- apps/demo/src/demo-config.ts
</read_first>

<action>
In `apps/demo/src/flow-animator.ts`:

1. Add import: `import { demoConfig } from './demo-config.js';`
2. Remove the module-level constant: `const FLASH_DURATION = 500;`
3. Replace all references to `FLASH_DURATION` with `demoConfig.get('demo.FLASH_DURATION')`.

Specifically, in `flashClass()`:
```typescript
function flashClass(el: Element, cls: string): void {
  el.classList.add(cls);
  setTimeout(() => el.classList.remove(cls), demoConfig.get('demo.FLASH_DURATION'));
}
```

This means every flash after a config change uses the new value immediately.
</action>

1. Add import: `import { demoConfig } from './demo-config.js';`
2. Remove the module-level constant: `const FLASH_DURATION_MS = 500;`
3. Replace all references to `FLASH_DURATION_MS` with `demoConfig.get('demo.FLASH_DURATION_MS')`.

Specifically, in the `flash()` method of the EdgeFlasher returned by `initTopologyEdges()`:
```typescript
flash(edgeId: string, cls: 'active' | 'amber' | 'blocked'): void {
  const color = cls === 'active' ? COLOR_ACTIVE : cls === 'amber' ? COLOR_AMBER : COLOR_BLOCKED;
  for (const suffix of ['-out', '-in']) {
    const line = lines.get(edgeId + suffix);
    if (!line) continue;
    try {
      line.setOptions({ color, size: 3 });
      setTimeout(() => {
        try { line.setOptions({ color: COLOR_RESTING, size: 2 }); } catch { /* best-effort */ }
      }, demoConfig.get('demo.FLASH_DURATION_MS'));
    } catch { /* best-effort */ }
  }
},
```

Do NOT modify COLOR_ACTIVE, COLOR_AMBER, COLOR_BLOCKED, COLOR_RESTING — those are visual constants, not behavioral magic numbers.
</action>

export function recordSignerRequest(record: SignerRequestRecord): void {
  const maxRecent = demoConfig.get('demo.MAX_RECENT_REQUESTS');
  const requests = [..._state.recentRequests, record];
  if (requests.length > maxRecent) {
    requests.splice(0, requests.length - maxRecent);

}
