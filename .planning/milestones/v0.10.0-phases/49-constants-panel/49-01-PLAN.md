# Plan 49-01: Mutable Demo Config Object

<objective>
Create the mutable configuration object that holds all protocol magic numbers with their defaults, supports per-constant get/set/reset, tracks which values differ from defaults, and provides session-scoped persistence (survives panel close/reopen, resets on page reload).
</objective>

<action>
Create `apps/demo/src/demo-config.ts` with:

1. **ConstantDef interface:**
```typescript
export interface ConstantDef {
  key: string;           // unique ID e.g. 'core.REPLAY_WINDOW_SECONDS'
  label: string;         // human-readable e.g. 'Replay Window'
  defaultValue: number;  // original module-level const value
  currentValue: number;  // live mutable value
  unit: string;          // 'ms' | 's' | 'count' | 'bytes' | 'px'
  min: number;           // slider min
  max: number;           // slider max
  step: number;          // slider step
  pkg: string;           // package grouping: 'core' | 'runtime' | 'services' | 'acl' | 'shell' | 'demo'
  domain: string;        // domain grouping: 'timeouts' | 'sizes' | 'ui-timing' | 'protocol'
  editable: boolean;     // false for protocol constants
  description: string;   // tooltip/hover text
}
```

2. **CONSTANT_DEFS array** with all constants from the inventory:

Editable constants (11 behavioral):
- `core.REPLAY_WINDOW_SECONDS` — default 30, unit 's', min 1, max 300, step 1, pkg 'core', domain 'timeouts'
- `runtime.RING_BUFFER_SIZE` — default 100, unit 'count', min 1, max 1000, step 1, pkg 'runtime', domain 'sizes'
- `shim.REQUEST_TIMEOUT_MS` — default 5000, unit 'ms', min 100, max 60000, step 100, pkg 'shim', domain 'timeouts', editable false (D-13: shim not affected, display only)
- `services.DEFAULT_EOSE_TIMEOUT_MS` — default 15000, unit 'ms', min 1000, max 60000, step 500, pkg 'services', domain 'timeouts'
- `services.EOSE_FALLBACK_MS` — default 15000, unit 'ms', min 1000, max 60000, step 500, pkg 'services', domain 'timeouts'
- `services.DEFAULT_MAX_PER_WINDOW` — default 100, unit 'count', min 1, max 1000, step 1, pkg 'services', domain 'sizes'
- `acl.DEFAULT_QUOTA` — default 524288, unit 'bytes', min 1024, max 10485760, step 1024, pkg 'acl', domain 'sizes'
- `demo.FLASH_DURATION` — default 500, unit 'ms', min 50, max 5000, step 50, pkg 'demo', domain 'ui-timing'
- `demo.FLASH_DURATION_MS` — default 500, unit 'ms', min 50, max 5000, step 50, pkg 'demo', domain 'ui-timing'

3. **DemoConfig class** with:
- `private _defs: Map<string, ConstantDef>` — keyed by constant key
- `get(key: string): number` — returns currentValue
- `set(key: string, value: number): void` — updates currentValue, clamps to min/max
- `reset(key: string): void` — resets one to default
- `resetAll(): void` — resets all to defaults
- `isModified(key: string): boolean` — currentValue !== defaultValue
- `getModifiedKeys(): string[]` — all keys where isModified is true
- `getAllDefs(): ConstantDef[]` — full list for rendering
- `getByPackage(): Map<string, ConstantDef[]>` — grouped by pkg
- `getByDomain(): Map<string, ConstantDef[]>` — grouped by domain
- `subscribe(callback: (key: string, value: number) => void): () => void` — change notifications

4. **Singleton export:**
```typescript
export const demoConfig = new DemoConfig(CONSTANT_DEFS);
```

The class stores all state in memory (Map). Page reload destroys it (session-only per D-10).
