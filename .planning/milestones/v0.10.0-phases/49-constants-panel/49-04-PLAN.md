# Plan 49-04: Runtime Config Overrides and Final Integration

<objective>
Thread runtime-level config overrides (REPLAY_WINDOW_SECONDS, RING_BUFFER_SIZE) through RuntimeAdapter so the demo can modify them at runtime. Wire the constants panel into main.ts bootstrap, connect the demoConfig change listener to re-render the panel, and ensure the full end-to-end flow works.
</objective>

<action>
In `packages/runtime/src/types.ts`, add a new optional interface to RuntimeAdapter:

```typescript
/** Optional runtime configuration overrides. When provided, the runtime reads
 *  these instead of the module-level defaults. All fields are optional —
 *  unset fields use the built-in defaults. */
export interface RuntimeConfigOverrides {
  /** Override REPLAY_WINDOW_SECONDS (default: 30). */
  replayWindowSeconds?: number;
  /** Override RING_BUFFER_SIZE (default: 100). */
  ringBufferSize?: number;
}
```

Add to the RuntimeAdapter interface:
```typescript
/** Optional runtime behavior overrides — demo/debug use only. */
getConfigOverrides?(): RuntimeConfigOverrides;
```

This is entirely optional. Existing RuntimeAdapter implementations are unaffected.

<task id="49-04-02">
<title>Read config overrides in runtime.ts and event-buffer.ts</title>
<read_first>
- packages/runtime/src/runtime.ts
- packages/runtime/src/event-buffer.ts
- packages/core/src/constants.ts
</read_first>

<action>
In `packages/runtime/src/runtime.ts`:

1. In `createRuntime()`, read the config overrides at the start:
```typescript
const configOverrides = hooks.getConfigOverrides?.() ?? {};
```

2. Where `REPLAY_WINDOW_SECONDS` is currently imported from `@napplet/core` and used for replay detection, use the override if present:
```typescript
const replayWindow = configOverrides.replayWindowSeconds ?? REPLAY_WINDOW_SECONDS;
```
Apply this in the `createReplayDetector()` call or wherever the replay window is checked. Search for all usages of `REPLAY_WINDOW_SECONDS` in runtime.ts and replace with the local `replayWindow` variable.

3. For `RING_BUFFER_SIZE`: the event buffer is created via `createEventBuffer()`. The buffer size needs to be configurable. Modify the flow:

In `packages/runtime/src/event-buffer.ts`:
- Change `createEventBuffer` to accept an optional `bufferSize?: number` parameter:
```typescript
export function createEventBuffer(
  sendToNapplet: SendToNapplet,
  sessionRegistry: SessionRegistry,
  enforce: (pubkey: string, capability: Capability) => EnforceResult,
  subscriptions: Map<string, SubscriptionEntry>,
  bufferSize?: number,
): EventBuffer {
  const maxSize = bufferSize ?? RING_BUFFER_SIZE;
  const buffer: NostrEvent[] = [];
  // ...
  bufferAndDeliver(event, senderId) {
    if (buffer.length >= maxSize) buffer.shift();
    // ...
  }
}
```

Back in `runtime.ts`, pass the override:
```typescript
const eventBuffer = createEventBuffer(
  sendToNapplet, sessionRegistry, enforce, subscriptions,
  configOverrides.ringBufferSize,
);
```

Note: `getConfigOverrides` is called once at runtime creation. For the demo to pick up changes dynamically, the demo can recreate the runtime (which it does on page reload anyway) OR the runtime can read the override lazily. Since per D-10 edits are session-only and D-12 says "config values instead of module-level const for overridable constants", the simplest approach that gives immediate effect is to make `getConfigOverrides()` a function that reads live values. The demo's RuntimeAdapter implementation will call `demoConfig.get()` inside `getConfigOverrides()`, so each time the runtime checks (which happens at message time for replay), it gets the current value.

For replay: `createReplayDetector()` likely accepts the window as a parameter. If it doesn't, modify it to accept one. If it uses REPLAY_WINDOW_SECONDS internally, thread the override through.

For buffer: the buffer check happens on `bufferAndDeliver()` call. Change `maxSize` to be read lazily:
```typescript
const getMaxSize = () => bufferSize ?? RING_BUFFER_SIZE;
// in bufferAndDeliver:
if (buffer.length >= getMaxSize()) buffer.shift();
```
Actually, simpler: pass a getter function instead of a static number:
```typescript
export function createEventBuffer(
  sendToNapplet: SendToNapplet,
  sessionRegistry: SessionRegistry,
  enforce: (pubkey: string, capability: Capability) => EnforceResult,
  subscriptions: Map<string, SubscriptionEntry>,
  getBufferSize?: () => number,
): EventBuffer {
  const buffer: NostrEvent[] = [];
  // ...
  bufferAndDeliver(event, senderId) {
    const maxSize = getBufferSize?.() ?? RING_BUFFER_SIZE;
    if (buffer.length >= maxSize) buffer.shift();
```

Then in runtime.ts:
```typescript
const eventBuffer = createEventBuffer(
  sendToNapplet, sessionRegistry, enforce, subscriptions,
  hooks.getConfigOverrides ? () => hooks.getConfigOverrides!().ringBufferSize ?? RING_BUFFER_SIZE : undefined,
);
```

This gives true live override behavior.
</action>
