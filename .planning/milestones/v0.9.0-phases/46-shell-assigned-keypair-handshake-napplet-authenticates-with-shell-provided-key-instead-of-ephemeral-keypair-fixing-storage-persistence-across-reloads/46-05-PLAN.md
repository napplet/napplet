# Plan 46-05: Aggregate Hash Verification and Caching

## Objective

Add shell-side aggregate hash verification to the REGISTER handler. When a napplet registers, the shell verifies its declared aggregate hash by computing the hash from fetched files (via the HashVerifierAdapter). Verification results are cached by manifest event ID so recomputation is skipped on subsequent loads with the same manifest. Hash mismatch triggers the onHashMismatch callback for user-visible warnings.

```typescript
    // ─── Aggregate hash verification (VERIFY-01, VERIFY-02, VERIFY-03) ──────
    if (aggregateHash && hooks.hashVerifier) {
      // Check verification cache first (D-05: cached by manifest event ID)
      // Look up manifest for this napplet to get event ID
      const cachedManifest = manifestCache.get('', dTag);
      const manifestEventId = cachedManifest ? `${dTag}:${aggregateHash}` : null;

      if (manifestEventId && manifestCache.hasVerification(manifestEventId)) {
        // Use cached verification result
        const cached = manifestCache.getVerification(manifestEventId)!;
        if (!cached.valid) {
          // Previously verified as mismatched — warn again
          hooks.onHashMismatch?.(dTag, aggregateHash, cached.aggregateHash);
        }
      } else {
        // Perform fresh verification
        try {
          const computed = await hooks.hashVerifier.computeHash(
            windowId, // Shell adapter can resolve the napplet URL from windowId
            [], // File list from manifest — shell adapter implementation detail
          );

          if (computed !== null) {
            const valid = computed === aggregateHash;
            const verificationEntry: VerificationCacheEntry = {
              aggregateHash: computed,
              valid,
              verifiedAt: Date.now(),
            };

            // Cache the result
            if (manifestEventId) {
              manifestCache.setVerification(manifestEventId, verificationEntry);
            }

            if (!valid) {
              // VERIFY-02: Hash mismatch — notify host for user warning
              hooks.onHashMismatch?.(dTag, aggregateHash, computed);
              // Note: We proceed with registration (D-04 says "user warning", not rejection)
              // The napplet is loaded but with a trust warning
            }
          }
          // If computed is null, verification was not possible (offline, CORS, etc.)
          // Proceed without verification — fail open for usability
        } catch {
          // Verification error — proceed without blocking (best-effort verification)
        }
      }
    }
```

Also add the import for the verification type at the top:


This needs to be added to the existing import from `./types.js` — merge it with the other type imports from that module.
