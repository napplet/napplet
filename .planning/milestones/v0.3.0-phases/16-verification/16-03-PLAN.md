# Plan 16-03: Integration Test for Shell-Runtime-ACL-Core Chain

## Objective

Create an integration test that verifies the full dependency chain: @napplet/shell (browser adapter) delegates to @napplet/runtime (protocol engine), which uses @napplet/acl (capability checks) and @napplet/core (shared types). This proves the four-package architecture works end-to-end.

## Context

Phase 14 makes shell a thin adapter over runtime. TST-04 requires an integration test verifying the full chain. This test lives in `tests/unit/` (the root vitest config includes `tests/unit/**/*.test.ts`) and imports from all four packages.

Unlike the e2e Playwright tests (which test the protocol in a real browser with iframes), this integration test runs in Node.js and verifies that the package layering is correct — shell creates a runtime, runtime uses acl for checks, and all types come from core.

- Create an ACL state using `createState('restrictive')` from @napplet/acl

- Verify that when a message requiring relay:write is processed, it is denied
