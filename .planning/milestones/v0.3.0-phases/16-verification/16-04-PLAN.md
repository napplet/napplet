# Plan 16-04: Existing E2E Test Suite Green with New Package Structure

## Objective

Ensure all 122 existing Playwright e2e tests pass after the four-package restructure (Phases 12-15). Fix any import path changes needed in test files, test helpers, test fixtures, and the test harness.

## Context

TST-01 requires all 122 existing Playwright e2e tests pass without modification, or with import-path-only changes. The restructure in Phases 12-14 changes where types and functions are exported from:

- Types previously from `@napplet/shell` (NostrEvent, Capability, BusKind) may now come from `@napplet/core`
- Enforce functions may now come from `@napplet/runtime`
- Shell's public API (createShellBridge, ShellHooks) stays in `@napplet/shell`

The test harness (`tests/e2e/harness/`) imports from `@napplet/shell` to create the shell host page. Test helpers (`tests/helpers/`) import ShellHooks and other types from `@napplet/shell`. Test napplets import from `@napplet/shim`.

Changes should be limited to import paths. Test logic and assertions must remain identical.

<action>
Read the test harness source. For every import from `@napplet/shell`:

1. If the import is a **type** that moved to `@napplet/core` (NostrEvent, NostrFilter, Capability, BusKind, etc.):
   - Change the import to `@napplet/core`
   - Example: `import type { NostrEvent } from '@napplet/shell'` → `import type { NostrEvent } from '@napplet/core'`

2. If the import is a **function or value** that stayed in `@napplet/shell` (createShellBridge, originRegistry, nappKeyRegistry, aclStore, audioManager, etc.):
   - Keep the import from `@napplet/shell`

3. If the import is a **function** that moved to `@napplet/runtime` (createEnforceGate, resolveCapabilities, etc.):
   - Change the import to `@napplet/runtime`

4. If `@napplet/shell` re-exports moved items for backward compatibility (check shell/src/index.ts), the existing imports may still work. In that case, update imports anyway to use the canonical source — this validates the package structure.

**Important:** Shell's package.json (Phase 14, SHELL-07) lists @napplet/runtime + @napplet/core as dependencies, so the test harness should already have access to all packages through the monorepo workspace.

Check if `tests/e2e/harness/package.json` lists its dependencies. If it only depends on `@napplet/shell`, it may need `@napplet/core` added (since core types will be imported directly). However, if shell re-exports core types, this may not be needed.
</action>
