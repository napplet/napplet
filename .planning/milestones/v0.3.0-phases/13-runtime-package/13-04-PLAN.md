# Plan 13-04: Implement createRuntime Factory with Full Message Dispatch

<objective>
Implement the `createRuntime(hooks: RuntimeHooks)` factory function that creates the complete protocol engine — handling all five NIP-01 verbs (EVENT, REQ, CLOSE, COUNT, AUTH), subscription lifecycle, AUTH handshake, signer proxying, and shell command routing. This is the centerpiece of @napplet/runtime.
</objective>

<context>
Plans 13-01 through 13-03 have scaffolded the package and moved the supporting modules:
- RuntimeHooks interface (types.ts)
- enforce.ts (ACL gate)
- napp-key-registry.ts (identity registry)
- acl-state.ts (ACL persistence wrapper)
- manifest-cache.ts (manifest persistence)
- replay.ts (replay detection)
- event-buffer.ts (ring buffer + subscription delivery)

Now the core dispatch logic from shell-bridge.ts's `createShellBridge()` must be extracted into `createRuntime()`. The runtime wires up all the sub-modules and handles every verb. The key difference from shell-bridge.ts: instead of `sourceWindow.postMessage(msg, '*')`, the runtime calls `hooks.sendToNapplet(windowId, msg)`.

Shell-specific concerns that do NOT go into the runtime:
- originRegistry (Window reference lookup) — the runtime receives a windowId directly
- audioManager (Web Audio, CustomEvent) — stays in shell
- state-proxy (localStorage for napp state) — the runtime delegates to RuntimeStatePersistence hooks

State-related request handling IS a protocol concern (the runtime dispatches state operations), but the actual storage is behind the hooks.
</context>

<action>
Create `packages/runtime/src/state-handler.ts` — extracts the state request handling from shell's state-proxy.ts, delegating storage to `RuntimeStatePersistence` hooks.

Key differences from shell's state-proxy.ts:
1. No `localStorage` calls — uses `RuntimeStatePersistence` methods
2. No `Window.postMessage` — uses `SendToNapplet` to send responses
3. No `window` or DOM references
4. Receives dependencies via parameters (not module-level singletons)

```ts
import type { NostrEvent } from '@napplet/core';
import { BusKind } from '@napplet/core';
import type { SendToNapplet, RuntimeStatePersistence } from './types.js';
import type { NappKeyRegistry } from './napp-key-registry.js';
import type { AclStateContainer } from './acl-state.js';

function scopedKey(pubkey: string, dTag: string, aggregateHash: string, userKey: string): string {
  return `napp-state:${pubkey}:${dTag}:${aggregateHash}:${userKey}`;
}

function sendResponse(
  sendToNapplet: SendToNapplet,
  windowId: string,
  correlationId: string,
  tags: string[][],
): void {
  const responseEvent: Partial<NostrEvent> = {
    kind: BusKind.INTER_PANE,
    pubkey: '',
    created_at: Math.floor(Date.now() / 1000),
    tags: [['t', 'napp:state-response'], ['id', correlationId], ...tags],
    content: '',
    id: '',
    sig: '',
  };
  sendToNapplet(windowId, ['EVENT', '__shell__', responseEvent]);
}

function sendError(
  sendToNapplet: SendToNapplet,
  windowId: string,
  correlationId: string,
  reason: string,
): void {
  sendResponse(sendToNapplet, windowId, correlationId, [['error', reason]]);
}

/**
 * Handle a state request from a napplet.
 * Routes to the appropriate operation (get, set, remove, clear, keys) based on topic.
 */
export function handleStateRequest(
  windowId: string,
  event: NostrEvent,
  sendToNapplet: SendToNapplet,
  nappKeyRegistry: NappKeyRegistry,
  aclState: AclStateContainer,
  statePersistence: RuntimeStatePersistence,
): void {
  const topic = event.tags?.find((t) => t[0] === 't')?.[1];
  const key = event.tags?.find((t) => t[0] === 'key')?.[1];
  const correlationId = event.tags?.find((t) => t[0] === 'id')?.[1] ?? '';

  const pubkey = nappKeyRegistry.getPubkey(windowId);
  if (!pubkey) { sendError(sendToNapplet, windowId, correlationId, 'auth-required: not registered'); return; }
  const entry = nappKeyRegistry.getEntry(pubkey);
  if (!entry) { sendError(sendToNapplet, windowId, correlationId, 'auth-required: no entry'); return; }

  const { dTag, aggregateHash } = entry;
  const prefix = `napp-state:${pubkey}:${dTag}:${aggregateHash}:`;

  switch (topic) {
    case 'shell:state-get': {
      if (!key) { sendError(sendToNapplet, windowId, correlationId, 'missing key tag'); return; }
      const sk = scopedKey(pubkey, dTag, aggregateHash, key);
      const result = statePersistence.get(sk);
      sendResponse(sendToNapplet, windowId, correlationId, [
        ['value', result ?? ''], ['found', result !== null ? 'true' : 'false'],
      ]);
      break;
    }
    case 'shell:state-set': {
      if (!key) { sendError(sendToNapplet, windowId, correlationId, 'missing key tag'); return; }
      const value = event.tags?.find((t) => t[0] === 'value')?.[1] ?? '';
      const quota = aclState.getStateQuota(pubkey, dTag, aggregateHash);
      const sk = scopedKey(pubkey, dTag, aggregateHash, key);
      const newWriteBytes = new TextEncoder().encode(sk + value).length;
      const existingBytes = statePersistence.calculateBytes(prefix, key);
      if (existingBytes + newWriteBytes > quota) {
        sendError(sendToNapplet, windowId, correlationId, `quota exceeded: napp state limit is ${quota} bytes`);
        return;
      }
      const success = statePersistence.set(sk, value);
      if (success) {
        sendResponse(sendToNapplet, windowId, correlationId, [['ok', 'true']]);
      } else {
        sendError(sendToNapplet, windowId, correlationId, 'state write failed');
      }
      break;
    }
    case 'shell:state-remove': {
      if (!key) { sendError(sendToNapplet, windowId, correlationId, 'missing key tag'); return; }
      const sk = scopedKey(pubkey, dTag, aggregateHash, key);
      statePersistence.remove(sk);
      sendResponse(sendToNapplet, windowId, correlationId, [['ok', 'true']]);
      break;
    }
    case 'shell:state-clear': {
      statePersistence.clear(prefix);
      sendResponse(sendToNapplet, windowId, correlationId, [['ok', 'true']]);
      break;
    }
    case 'shell:state-keys': {
      const userKeys = statePersistence.keys(prefix);
      sendResponse(sendToNapplet, windowId, correlationId, userKeys.map(k => ['key', k]));
      break;
    }
    default:
      sendError(sendToNapplet, windowId, correlationId, `unknown state topic: ${topic}`);
      break;
  }
}

/**
 * Remove all state entries for a napp identity.
 * Used during napp cleanup when a window is closed.
 */
export function cleanupNappState(
  pubkey: string,
  dTag: string,
  aggregateHash: string,
  statePersistence: RuntimeStatePersistence,
): void {
  const prefix = `napp-state:${pubkey}:${dTag}:${aggregateHash}:`;
  statePersistence.clear(prefix);
}
```

Note: `TextEncoder` is available in both Node.js and browsers — it is NOT a DOM-only API. It's part of the WHATWG Encoding standard and available in all modern JS environments. No "DOM" lib needed in tsconfig.

<action>
Create `packages/runtime/src/runtime.ts` — the main factory that creates the protocol engine by wiring up all sub-modules and implementing the full NIP-01 verb dispatch.

The public `Runtime` interface:

```ts
import type { NostrEvent, NostrFilter, Capability } from '@napplet/core';
import type { RuntimeHooks, ConsentRequest, ConsentHandler } from './types.js';
import type { NappKeyRegistry } from './napp-key-registry.js';
import type { AclStateContainer } from './acl-state.js';
import type { ManifestCache } from './manifest-cache.js';

/**
 * The napplet protocol engine — handles all NIP-01 verb dispatch,
 * AUTH handshake, ACL enforcement, subscription lifecycle.
 */
export interface Runtime {
  /**
   * Handle an incoming NIP-01 message from a napplet.
   * The caller is responsible for identifying the source (windowId).
   *
   * @param windowId - The identifier of the napplet that sent the message
   * @param msg - The raw NIP-01 message array
   */
  handleMessage(windowId: string, msg: unknown[]): void;

  /**
   * Send a NIP-42 AUTH challenge to a napplet window.
   *
   * @param windowId - The window identifier
   */
  sendChallenge(windowId: string): void;

  /**
   * Inject a shell-originated event into subscription delivery.
   *
   * @param topic - The event topic tag value
   * @param payload - The event content
   */
  injectEvent(topic: string, payload: unknown): void;

  /** Destroy the runtime, persisting state and clearing all internal state. */
  destroy(): void;

  /** Register a handler for consent requests on destructive signing kinds. */
  registerConsentHandler(handler: ConsentHandler): void;

  /** Access the identity registry (for shell adapter to read napp state). */
  readonly nappKeyRegistry: NappKeyRegistry;

  /** Access the ACL state container. */
  readonly aclState: AclStateContainer;

  /** Access the manifest cache. */
  readonly manifestCache: ManifestCache;
}
```

Factory implementation — `export function createRuntime(hooks: RuntimeHooks): Runtime`:

The factory:
1. Creates instances of all sub-modules using the hooks:
   - `createNappKeyRegistry(hooks.onPendingUpdate)`
   - `createAclState(hooks.aclPersistence)`
   - `createManifestCache(hooks.manifestPersistence)`
   - `createReplayDetector()`
   - `createEnforceGate({ checkAcl, resolveIdentity, onAclCheck })`
   - `createEventBuffer(hooks.sendToNapplet, nappKeyRegistry, enforce, subscriptions)`

2. Implements `handleMessage(windowId, msg)`:
   - Validates msg is array with length >= 2
   - If verb is 'AUTH' → `handleAuth(msg, windowId)`
   - If nappKeyRegistry.getPubkey(windowId) is undefined → queue in pendingAuthQueue
   - Otherwise → `dispatchVerb(verb, msg, windowId)`

3. Implements all verb handlers (matching current shell-bridge.ts behavior exactly):
   - `handleAuth(msg, windowId)` — NIP-42 challenge-response, signature verification, napp update detection
   - `handleEvent(msg, windowId)` — replay check, enforce, dispatch to signer/hotkey/inter-pane/default
   - `handleReq(msg, windowId)` — subscribe, replay cached events, query relay pool and worker relay
   - `handleClose(msg, windowId)` — remove subscription
   - `handleCount(msg, windowId)` — count matching events in buffer

4. Key differences from shell-bridge.ts:
   - All `sourceWindow.postMessage(msg, '*')` replaced with `hooks.sendToNapplet(windowId, msg)`
   - All `originRegistry.getIframeWindow(windowId)` calls removed — sendToNapplet is the only output path
   - All `localStorage` calls replaced with persistence hooks
   - All `window.` references removed
   - All `crypto.randomUUID()` replaced with `hooks.crypto.randomUUID()`
   - `hooks.relayPool` uses the RuntimeRelayPoolHooks interface (subscribe returns handle, not rxjs-style)
   - Audio commands (`shell:audio-*`) are forwarded to an optional `hooks.onAudioCommand?.(event, windowId)` callback or simply buffered/delivered as inter-pane events. The runtime does NOT manage audio state — that stays in shell. Audio events are just inter-pane events from the runtime's perspective.
   - State requests (`shell:state-*`) delegated to `handleStateRequest()` from state-handler.ts
   - Shell commands (`shell:acl-*`, `shell:relay-*`, `shell:create-window`, `shell:send-dm`) use aclState, relayConfig hooks, windowManager hooks, dm hooks

5. `handleMessage` signature is `(windowId: string, msg: unknown[])` — the caller (shell adapter) does the origin lookup and passes the windowId. No MessageEvent needed.

Important implementation detail for handleReq: The current shell-bridge.ts handleReq uses `hooks.relayPool.getRelayPool()` which returns an rxjs-style pool with `.subscription().subscribe()`. The RuntimeRelayPoolHooks uses a simpler callback-based `subscribe(filters, callback)` that returns `{ unsubscribe() }`. The adapter implementation is the same but the API shape is different — adapt the handleReq logic to use the new interface.

For handleAuth — the current implementation calls `hooks.crypto.verifyEvent(authEvent)`. The runtime version calls the same on `hooks.crypto.verifyEvent(authEvent)`. For napp updates with `banner` behavior, the runtime calls `nappKeyRegistry.setPendingUpdate()` which triggers the notifier (instead of CustomEvent).

For handleSignerRequest — the signer is obtained from `hooks.auth.getSigner()` and the consent check uses `aclState.requiresPrompt(kind)` and the registered `_consentHandler`.

IMPORTANT: `TextEncoder` is needed for state quota calculation in the state-handler. It is available in Node.js 12+ and all modern browsers without DOM lib. However, verify it compiles with only `ES2022` lib.

```ts
function byteLength(str: string): number {
  let bytes = 0;
  for (let i = 0; i < str.length; i++) {
    const c = str.charCodeAt(i);
    if (c < 0x80) bytes += 1;
    else if (c < 0x800) bytes += 2;
    else if (c < 0xd800 || c >= 0xe000) bytes += 3;
    else { i++; bytes += 4; } // surrogate pair
  }
  return bytes;
}
```
This avoids needing the DOM or WebWorker typings entirely.
