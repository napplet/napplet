# Plan 13-05: Full Build Verification and Cross-Package Type-Check

<objective>
Verify that @napplet/runtime builds cleanly, has no browser-specific imports, and type-checks in the full monorepo build. This is the final gate before Phase 14 wires the shell adapter.
</objective>

<context>
Plans 13-01 through 13-04 created the full @napplet/runtime package with:
- RuntimeHooks interface and all sub-hook types
- enforce.ts (ACL gate)
- napp-key-registry.ts (identity registry)
- acl-state.ts (ACL wrapper with persistence)
- manifest-cache.ts (manifest wrapper with persistence)
- replay.ts (replay detection)
- event-buffer.ts (ring buffer + delivery)
- state-handler.ts (state operations)
- runtime.ts (main factory with full dispatch)

This plan ensures everything integrates correctly and the package meets all RT-12/RT-13 criteria.
</context>

<tasks>

<task id="13-05-01" title="Verify no browser-specific imports in runtime package">
<read_first>
- packages/runtime/tsconfig.json
- packages/runtime/src/index.ts
</read_first>
<action>
Run a comprehensive scan for browser-specific APIs across all runtime source files:

```bash
# Check for DOM/browser API references
grep -rn 'window\.\|document\.\|localStorage\|sessionStorage\|postMessage\|MessageEvent\|Window\b\|HTMLElement\|CustomEvent\|addEventListener\|removeEventListener\|navigator\.\|location\.\|XMLHttpRequest\|fetch(' packages/runtime/src/

# Check for DOM type imports
grep -rn "from 'dom'\|/// <reference lib=\"dom\"" packages/runtime/src/
```

Both commands should return 0 results.

Also verify the tsconfig.json lib does NOT include DOM:
```bash
grep 'DOM' packages/runtime/tsconfig.json
```
Should return 0 results.

If `TextEncoder` caused a compilation issue (it requires DOM or WebWorker lib in TypeScript), verify it was replaced with a pure JS byte-counting function as described in Plan 13-04.

If any browser references are found, fix them:
- Replace `window.dispatchEvent` with notifier callbacks
- Replace `localStorage` with persistence hooks
- Replace `crypto.randomUUID()` with `hooks.crypto.randomUUID()`
- Replace `Window` type with `string` windowId
- Replace `MessageEvent` with `{ windowId: string, msg: unknown[] }` pattern
</action>
<acceptance_criteria>
- `grep -rnc 'window\.\|document\.\|localStorage\|sessionStorage\|MessageEvent' packages/runtime/src/ | grep -v ':0$'` returns empty output (all files report 0 matches)
- `grep -c 'DOM' packages/runtime/tsconfig.json` prints "0"
- `pnpm --filter @napplet/runtime type-check` succeeds (proves no DOM types needed)
</acceptance_criteria>
</task>

Verify that @napplet/runtime's package.json declares exactly the allowed dependencies:

```bash
node -e "
  const pkg = JSON.parse(require('fs').readFileSync('packages/runtime/package.json', 'utf8'));
  const deps = Object.keys(pkg.dependencies || {}).sort();
  console.log('Dependencies:', deps.join(', '));
  console.log('Count:', deps.length);
  const hasCore = deps.includes('@napplet/core');
  const hasAcl = deps.includes('@napplet/acl');

  console.log('Peer dependencies:', peerDeps.length === 0 ? 'none (correct)' : peerDeps.join(', '));
```

Expected output:
- Dependencies: @napplet/acl, @napplet/core
- Count: 2
