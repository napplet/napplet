## Objective

Wire up all cross-package dependencies, verify the full monorepo builds, and confirm that the dependency graph is correct: `@napplet/shell` depends on `@napplet/runtime` + `@napplet/core`, `@napplet/shim` depends on `@napplet/core`, and no circular dependencies exist. Verify the entire build chain works end-to-end.

## Context

After Plans 01 and 02, the source code is updated but the package infrastructure may not be wired correctly:
- `packages/shell/package.json` needs `@napplet/runtime` and `@napplet/core` as dependencies
- `tsconfig.json` references may need updating for the new package imports
- `turbo.json` build pipeline must account for the dependency order: core -> acl -> runtime -> shell (and core -> shim)
- The pnpm lockfile needs regenerating after dependency changes

Verify the dependency DAG is clean:

```
@napplet/core     -> (no @napplet/* deps)
@napplet/acl      -> (no @napplet/* deps)
@napplet/runtime  -> @napplet/core, @napplet/acl
@napplet/shell    -> @napplet/core, @napplet/runtime
@napplet/shim     -> @napplet/core
```

Check each package.json:
1. `packages/core/package.json` — no `dependencies` or `peerDependencies` containing `@napplet/*`
2. `packages/acl/package.json` — no `dependencies` or `peerDependencies` containing `@napplet/*`
3. `packages/runtime/package.json` — only `@napplet/core` and `@napplet/acl` in dependencies
4. `packages/shell/package.json` — only `@napplet/core` and `@napplet/runtime` in dependencies (NOT @napplet/acl directly — runtime handles ACL)
5. `packages/shim/package.json` — only `@napplet/core` in dependencies

If `@napplet/shell` depends on `@napplet/acl` directly (for aclStore which stays in shell), that may be acceptable but should be noted. The ideal is: shell -> runtime -> acl (indirect).

However, since `acl-store.ts` stays in shell and imports from `@napplet/acl`, shell MAY need a direct dep on `@napplet/acl`. Check the actual imports and add the dependency if needed.
</action>
<acceptance_criteria>
- `packages/core/package.json` does NOT contain `@napplet/` in dependencies (verified by: `grep -c '@napplet/' packages/core/package.json` returns 0 or only appears in non-dependency fields)
