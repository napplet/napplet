## Context

`@napplet/acl` is a zero-dependency, pure-function ACL module. All functions are state-in/result-out. It uses bitfield constants (`CAP_*`) for fast capability checks. It is consumed by `@napplet/runtime` (via `aclPersistence` hooks and internal state). Shell authors who want to inspect or manipulate ACL state use this package directly.

The shim README at `packages/shim/README.md` is the primary style reference:

**H1 and tagline:**
```
# @napplet/acl

> Pure, zero-dependency ACL module for the napplet protocol.
```

**Getting Started:**

Prerequisites section (short — pure lib, no host required):
- Note that no peer dependencies exist
- Note it works in browser, Node.js, and WASM contexts

How It Works (numbered list):
1. Create an `AclState` with a default policy (`createState('permissive')` or `createState('restrictive')`)
2. Use `check(state, identity, CAP_*)` to test whether an identity has a capability
3. Use `grant()`, `revoke()`, `block()`, `unblock()` to return new (immutable) states
4. Use `serialize()`/`deserialize()` to persist state — storage backend is the caller's responsibility

Installation:
```bash
npm install @napplet/acl
```

**Quick Start** — use the JSDoc @example from `packages/acl/src/index.ts` verbatim, adapting to `ts` code block:
```ts
import {
  createState, check, grant, revoke, block, unblock,
  CAP_RELAY_READ, CAP_SIGN_EVENT,
} from '@napplet/acl';

// Create state with restrictive default (deny unknown identities)
let state = createState('restrictive');

const id = { pubkey: 'abc...', dTag: 'chat', hash: 'ff00...' };

// Grant relay read access
state = grant(state, id, CAP_RELAY_READ);
check(state, id, CAP_RELAY_READ);  // true
check(state, id, CAP_SIGN_EVENT);  // false (not granted)

// Block the identity (overrides all caps)
state = block(state, id);
check(state, id, CAP_RELAY_READ);  // false (blocked)

// Unblock restores previous capabilities
state = unblock(state, id);
check(state, id, CAP_RELAY_READ);  // true (restored)
```

**API Reference:**

Section `### State Creation`
- `createState(policy?)` — param table: `policy | 'permissive' | 'restrictive' | Default policy for unknown identities. Defaults to 'permissive'` — Returns: `AclState` — code example showing `createState('restrictive')` and `createState()` (permissive default)

Section `### Capability Check`
- `check(state, identity, cap)` — param table: `state | AclState`, `identity | Identity`, `cap | number` — Returns: `boolean` — note the 3-step decision tree: (1) no entry → defaultPolicy, (2) blocked → false, (3) bitfield test. Code example from check.ts JSDoc.
- `toKey(identity)` — param table: `identity | Identity` — Returns: `string` — note it returns `'pubkey:dTag:hash'`. Code: `toKey({ pubkey: 'abc', dTag: 'chat', hash: 'ff00' }) // => 'abc:chat:ff00'`

Section `### Mutations`
Write an introductory sentence: "All mutations return a new `AclState`. The original is never modified."
- `grant(state, identity, cap)` — params, returns `AclState` — 1-line code example
- `revoke(state, identity, cap)` — params, returns `AclState` — 1-line code example
- `block(state, identity)` — params, returns `AclState` — note: blocked flag is orthogonal to caps; unblocking restores caps
- `unblock(state, identity)` — params, returns `AclState`
- `setQuota(state, identity, bytes)` — params, returns `AclState` — note default is 512 KB (`DEFAULT_QUOTA`)
- `getQuota(state, identity)` — params, returns `number` (bytes)

Section `### Serialization`
- `serialize(state)` — Returns: `string` (JSON) — purpose: pass the result to your storage backend
- `deserialize(json)` — Returns: `AclState` — note: returns fresh permissive state on parse failure

**Capability Constants** section (NOT under API Reference — separate `##` section):
Table with columns `| Constant | Value | Capability |`:
- `CAP_RELAY_READ` | `1 << 0` (1) | `relay:read` — subscribe to relay events
- `CAP_RELAY_WRITE` | `1 << 1` (2) | `relay:write` — publish to relays
- `CAP_CACHE_READ` | `1 << 2` (4) | `cache:read` — read from local cache
- `CAP_CACHE_WRITE` | `1 << 3` (8) | `cache:write` — write to local cache
- `CAP_HOTKEY_FORWARD` | `1 << 4` (16) | `hotkey:forward` — forward keyboard shortcuts
- `CAP_SIGN_EVENT` | `1 << 5` (32) | `sign:event` — request event signing
- `CAP_SIGN_NIP04` | `1 << 6` (64) | `sign:nip04` — NIP-04 encrypt/decrypt
- `CAP_SIGN_NIP44` | `1 << 7` (128) | `sign:nip44` — NIP-44 encrypt/decrypt
- `CAP_STATE_READ` | `1 << 8` (256) | `state:read` — read napplet-scoped state
- `CAP_STATE_WRITE` | `1 << 9` (512) | `state:write` — write napplet-scoped state
- `CAP_ALL` | `(1 << 10) - 1` (1023) | All capabilities granted
- `CAP_NONE` | `0` | No capabilities granted
- `DEFAULT_QUOTA` | `524288` | Default state storage quota (512 KB)

**Types** section:
```ts
import type { AclState, AclEntry, Identity } from '@napplet/acl';
```
Table with `| Type | Description |`:
- `AclState` | Complete ACL state — immutable, contains `defaultPolicy` and `entries` map
- `AclEntry` | Entry for one identity — `caps` (bitfield), `blocked` (flag), `quota` (bytes)
- `Identity` | Napplet identity composite key — `pubkey`, `dTag`, `hash`

**Integration note** (brief paragraph, per D-10):
Note that `@napplet/runtime` uses this package internally. The `RuntimeAclPersistence` hooks in `@napplet/runtime` call `serialize()`/`deserialize()` to save/load state. Shell authors who need to inspect or customize ACL behavior can use these functions directly.
