**YAML frontmatter** (required, agentskills.io format):
```yaml
---
name: build-napplet
description: Use when writing a napplet (sandboxed Nostr iframe app) using @napplet/shim — covers Vite project setup, NIP-5A manifest plugin, subscribe/publish/query relay API, nappStorage, window.nostr NIP-07 proxy, inter-pane events, and service discovery

```

**Body — write these sections in order:**

1. **Overview** paragraph: A napplet is a sandboxed iframe app that communicates with a host shell via postMessage using NIP-01 wire format. The shim (`@napplet/shim`) provides the full client-side API — relay subscriptions, event signing proxy, scoped storage, and inter-pane messaging. The iframe runs without `allow-same-origin`; all host access is proxied over postMessage.

2. **Prerequisites** section:
   - Node.js 18+, pnpm (or npm/yarn)
   - A Vite-based project (`pnpm create vite`)
   - A host shell running `@napplet/shell` (or a test harness)

3. **Step 1 — Install dependencies**:
```bash
pnpm add @napplet/shim
pnpm add -D @napplet/vite-plugin
```

4. **Step 2 — Configure the Vite plugin** — show `vite.config.ts` with `nip5aManifest({ nappType })`. Use exact option names from `Nip5aManifestOptions` in `packages/vite-plugin/src/index.ts` (`nappType` required, `requires` optional with comment):
```ts
import { defineConfig } from 'vite';
import { nip5aManifest } from '@napplet/vite-plugin';

export default defineConfig({
  plugins: [
    nip5aManifest({
      nappType: 'my-napplet',
      // requires: ['audio', 'notifications'], // optional: declared service deps
    }),
  ],
});
```
Note: set `VITE_DEV_PRIVKEY_HEX` env var for signed manifests in CI. Dev builds work without it.

5. **Step 3 — Subscribe to relay events** — import `subscribe` from `@napplet/shim`, show realistic filter `{ kinds: [1], authors: [myPubkey], limit: 20 }`, show `onEvent`, `onEose` callbacks, show `sub.close()` teardown.

6. **Step 4 — Publish an event** — import `publish` from `@napplet/shim`, call with `EventTemplate` (kind, content, tags, created_at). Note: signing is delegated to the shell signer — the napplet never holds private keys. Show `await publish({...})`.

7. **Step 5 — Query cached events** — show `query()` returning `Promise<NostrEvent[]>` from the shell's local cache. Example: `const events = await query({ kinds: [1], limit: 50 });`

8. **Step 6 — Use nappStorage** — import `nappStorage` from `@napplet/shim`. Show `setItem(key, value)`, `getItem(key)` (returns `string | null`), `removeItem(key)`. Note: storage is scoped by `nappType:aggregateHash` — different napplet versions have isolated storage. Show JSON.stringify/parse pattern for objects.

9. **Step 7 — Use window.nostr (NIP-07 proxy)** — explain that `@napplet/shim` installs `window.nostr` automatically on import. Show `getPublicKey()` and `signEvent()`. Note: the private key never leaves the shell — signing calls are proxied.

10. **Step 8 — Inter-pane events (emit / on)** — import `emit` and `on` from `@napplet/shim`. Show `emit('profile:open', [], JSON.stringify({ pubkey }))` for broadcasting. Show `on('profile:open', (payload) => {...})` for subscribing. Show `sub.close()` for teardown.

11. **Step 9 — Service discovery** — import `discoverServices`, `hasService`, `hasServiceVersion` from `@napplet/shim`. Show `await discoverServices()` returning array of `{ name, version, description }`. Show `await hasService('audio')` guard pattern. Show `await hasServiceVersion('audio', '1.0.0')` for version-specific use.

12. **Common pitfalls** section with at least these items:
    - Do not call `window.nostr` before `@napplet/shim` is imported — it is installed synchronously at module load, but signer calls are async.
    - `nappStorage` is scoped by version hash — clearing storage in one build version does not affect another.
    - Do not use `localStorage` directly — without `allow-same-origin` it will throw `SecurityError`.
    - `publish()` returns `Promise<NostrEvent>` — always `await` it. Errors surface as promise rejections.
    - `query()` resolves after EOSE — it is a one-shot snapshot, not a live stream. Use `subscribe()` for live updates.
    - `discoverServices()` results are session-cached. Refresh requires a page reload.

All code blocks must be TypeScript. Use realistic examples (not abstract placeholders). No framework dependencies (no Svelte/React/Vue).
