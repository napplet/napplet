# Plan 24-04: Update @napplet/vite-plugin README

## Objective

Update `packages/vite-plugin/README.md` to document the `requires` option added in v0.4.0 (Phase 22). This includes: a dedicated "Service Dependencies" section, the `requires` field in the `Nip5aManifestOptions` interface docs, and the HTML output that gets injected.

## Context

The current vite-plugin README documents `nappType` but the `Nip5aManifestOptions` interface also has `requires?: string[]` (added in Phase 22). The plugin injects a `<meta name="napplet-requires">` tag when `requires` is set, and adds `["requires", "service-name"]` tags to the kind 35128 manifest event. This is entirely absent from the README.

Key facts from `packages/vite-plugin/src/index.ts`:
- `Nip5aManifestOptions.requires?: string[]` — optional array of service name strings
- In `transformIndexHtml()`: injects `<meta name="napplet-requires" content="audio,notifications">` when requires is set (line 88-100)
- In `closeBundle()`: adds `['requires', name]` tags to the kind 35128 manifest event (line 132-133)
- Console log: `[nip5a-manifest] {nappType}: requires [audio, notifications]` (line 191-193)

## Service Dependencies

Use the `requires` option when your napplet needs specific shell services (like audio playback or push notifications) to function correctly.

```ts
// vite.config.ts
import { defineConfig } from 'vite';
import { nip5aManifest } from '@napplet/vite-plugin';

export default defineConfig({
  plugins: [
    nip5aManifest({
      nappType: 'my-music-app',
      requires: ['audio', 'notifications'],
    }),
  ],
});
```

### What gets injected

With `requires: ['audio', 'notifications']`, the plugin injects into your HTML `<head>`:

```html
<meta name="napplet-aggregate-hash" content="">
<meta name="napplet-napp-type" content="my-music-app">
<meta name="napplet-requires" content="audio,notifications">
```

At build time (with `VITE_DEV_PRIVKEY_HEX` set), the manifest event also includes `requires` tags:

```json
{
  "kind": 35128,
  "tags": [
    ["d", "my-music-app"],
    ["x", "<sha256>", "index.js"],
    ["requires", "audio"],
    ["requires", "notifications"]
  ]
}
```

### Runtime compatibility checking

The host shell reads `<meta name="napplet-requires">` during napplet initialization and compares against registered services. Napplets can also check at runtime:

```ts
import { hasService } from '@napplet/shim';

if (!(await hasService('audio'))) {
  console.warn('Audio service not available — some features disabled');
}
```
```
