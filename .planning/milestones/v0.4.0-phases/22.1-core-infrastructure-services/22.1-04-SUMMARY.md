# Summary: Plan 22.1-04 — Runtime Migration

## What Was Done

Wired the runtime to dispatch to registered services with fallback to internal hooks:

1. **SIGNER_REQUEST dispatch**: `handleEvent` checks `serviceRegistry['signer']` before calling internal `handleSignerRequest`. If a signer service is registered, dispatches via `handleMessage(windowId, ['EVENT', event], sendCb)`.

2. **REQ dispatch**: `handleReq` checks for `serviceRegistry['relay']` or `serviceRegistry['relay-pool']` before the cache/relay pool fallback blocks. Service receives the full REQ message. Separate cache service receives a parallel REQ if only relay-pool (not relay) is registered.

3. **CLOSE dispatch**: `handleClose` forwards CLOSE to relay service when registered, then calls `hooks.relayPool?.untrackSubscription()`.

4. **Optional hooks**: `RuntimeHooks.relayPool` and `RuntimeHooks.cache` made optional. All 7 direct references updated to use optional chaining (`hooks.relayPool?.`, `hooks.cache?.`).

5. **Tests**: 7 new service dispatch tests added to `dispatch.test.ts`. Tests cover signer routing, relay routing, EOSE delivery, CLOSE forwarding, and service-only operation without hooks.
