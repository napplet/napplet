## Naming Convention Reference

Shell sub-interfaces (`*Hooks`) describe injection points the host *provides* (callbacks). Runtime sub-interfaces (`*Adapter`) describe what the protocol engine *requires* (environment abstraction contracts). This distinction is intentional and must be preserved.

| Package | Old Name | New Name |
|---------|----------|----------|
| runtime | `RuntimeHooks` | `RuntimeAdapter` (+ deprecated alias `RuntimeHooks`) |
| shell | `ShellHooks` | `ShellAdapter` (+ deprecated alias `ShellHooks`) |
| runtime | `RuntimeRelayPoolHooks` | `RelayPoolAdapter` |
| runtime | `RuntimeCacheHooks` | `CacheAdapter` |
| runtime | `RuntimeAuthHooks` | `AuthAdapter` |
| runtime | `RuntimeSigner` | `Signer` |
| runtime | `RuntimeConfigHooks` | `ConfigAdapter` |
| runtime | `RuntimeHotkeyHooks` | `HotkeyAdapter` |
| runtime | `RuntimeAclPersistence` | `AclPersistence` |
| runtime | `RuntimeManifestPersistence` | `ManifestPersistence` |
| runtime | `RuntimeStatePersistence` | `StatePersistence` |
| runtime | `RuntimeCryptoHooks` | `CryptoAdapter` |
| runtime | `RuntimeWindowManagerHooks` | `WindowManagerAdapter` |
| runtime | `RuntimeRelayConfigHooks` | `RelayConfigAdapter` |
| runtime | `RuntimeDmHooks` | `DmAdapter` |

Shell sub-interfaces below are **NOT renamed** (keep `*Hooks`):
- `RelayPoolHooks`, `AuthHooks`, `ConfigHooks`, `HotkeyHooks`, `CryptoHooks`, `DmHooks`, `RelayConfigHooks`, `WindowManagerHooks`
