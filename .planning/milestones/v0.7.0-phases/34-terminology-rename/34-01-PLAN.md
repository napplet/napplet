# Plan 34-01: Core Type and Topic Constant Renames

## Objective

Rename `NappKeyEntry` → `SessionEntry` in the two files that define it (runtime/types.ts and shell/types.ts), and rename the two topic string constants in core/topics.ts. These files have no internal dependencies that would break from the rename — their consumers are updated in later waves.

This wave establishes the new canonical definitions before any consumers are touched.

---

## Task 34-01-01: Rename NappKeyEntry → SessionEntry in runtime/src/types.ts

<read_first>
- `packages/runtime/src/types.ts` — contains the canonical `NappKeyEntry` definition at line ~335; also has `PendingUpdate` which references `NappKeyEntry` indirectly
</read_first>

<action>
In `packages/runtime/src/types.ts`:

1. Find the section comment `// ─── NappKeyEntry ──────────────────────────────────────────────────────────`
   Change it to: `// ─── SessionEntry ─────────────────────────────────────────────────────────`

2. Find the JSDoc comment above the interface:
   ```
   /**
    * Registry entry mapping a napp's pubkey to its runtime metadata.
    * Created after a successful NIP-42 AUTH handshake.
    */
   export interface NappKeyEntry {
   ```
   Replace with:
   ```
   /**
    * Registry entry mapping a napplet's pubkey to its runtime metadata.
    * Created after a successful NIP-42 AUTH handshake.
    */
   export interface SessionEntry {
   ```

3. Find the section comment `// ─── NappKeyEntry ──────` above the `PendingUpdate` interface and update the nearby JSDoc:
   ```
   /**
    * A pending napp update — raised when a napp reconnects with a different aggregateHash.
    */
   export interface PendingUpdate {
   ```
   Replace the JSDoc line: `* A pending napp update — raised when a napp reconnects with a different aggregateHash.`
   With: `* A pending napplet update — raised when a napplet reconnects with a different aggregateHash.`

4. Add a deprecated alias directly after the `SessionEntry` interface closing brace:
   ```ts
   /** @deprecated Use SessionEntry. Will be removed in v0.9.0. */
   export type NappKeyEntry = SessionEntry;
   ```

5. Find the `ManifestCacheEntry` JSDoc and update:
   `* A cached manifest entry for a verified napp build.` → `* A cached manifest entry for a verified napplet build.`

NOTE: Do NOT change `PendingUpdateNotifier` type or any other types in the file. Only change `NappKeyEntry` interface and the immediate JSDoc/comment lines that specifically say "napp" meaning "napplet".
</action>

<acceptance_criteria>
- `packages/runtime/src/types.ts` contains `export interface SessionEntry {`
- `packages/runtime/src/types.ts` contains `export type NappKeyEntry = SessionEntry;` with `@deprecated` JSDoc
- `packages/runtime/src/types.ts` does NOT contain `export interface NappKeyEntry {` (the original definition is gone, replaced by alias)
- `pnpm type-check` exits 0 (may show errors from consumers that haven't been updated yet — those are expected and will be fixed in Wave 3)
</acceptance_criteria>
