## Task 34-04-05: Update vite-plugin/src/index.ts (meta attribute fix + backward compat)

<read_first>
- `packages/vite-plugin/src/index.ts` — `Nip5aManifestOptions.nappType` is a config option (keep as-is); does NOT currently inject a `napplet-napp-type` meta tag (it injects `napplet-aggregate-hash` and optionally `napplet-requires`); the `nappType` option feeds into the manifest `d` tag only
- `packages/shim/src/index.ts` — queries `meta[name="napplet-type"]` (new) and `meta[name="napplet-napp-type"]` (backward compat fallback)
</read_first>

<action>
In `packages/vite-plugin/src/index.ts`:

1. The `nappType` option name: Per CONTEXT.md, this is a public API option that feeds into the Nostr `d` tag. It is NOT renamed in Phase 34 (the collision here is the word "napp" in `nappType` meaning "what kind of napp/napplet", but it's a config key, not a type identifier). Leave `nappType` option name as-is.

2. Check if the plugin currently injects a `napplet-napp-type` meta tag. Based on reading the file, it does NOT inject this meta tag — the shim injects it via Vite config in the napplet project's own `vite.config.ts`. The vite-plugin only injects `napplet-aggregate-hash` and `napplet-requires`.

   HOWEVER, CONTEXT.md D-16/D-17 says:
   - "napplet-napp-type" → "napplet-type" in vite-plugin injection
   - Vite plugin reads BOTH for backward compat
   
   Since the plugin does NOT currently inject `napplet-napp-type`, this task needs to ADD injection of the `napplet-type` meta tag:
   
   In the `transformIndexHtml()` function, after the `napplet-aggregate-hash` tag injection, add:
   ```ts
   {
     tag: 'meta',
     attrs: {
       name: 'napplet-type',
       content: options.nappType,
     },
     injectTo: 'head' as const,
   },
   // Backward compatibility: also inject old attribute name for one release cycle
   {
     tag: 'meta',
     attrs: {
       name: 'napplet-napp-type',
       content: options.nappType,
     },
     injectTo: 'head' as const,
   },
   ```

3. Update the `Nip5aManifestOptions` interface JSDoc:
   Current: `/** Napp type/dtag (e.g., 'feed', 'chat') */`
   Replace with: `/** Napplet type/dtag identifier (e.g., 'feed', 'chat'). Used as the NIP-5A 'd' tag and injected as napplet-type meta attribute. */`

4. Update all console.log messages that say "napp type":
   Current: `console.log(\`[nip5a-manifest] ${options.nappType}: manifest signed by...\`)`
   These are logging the nappType VALUE (e.g., "chat", "feed"), not the word "napp". Leave as-is — these are fine.
</action>
