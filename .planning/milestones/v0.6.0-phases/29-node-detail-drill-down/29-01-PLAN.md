  <action>
    Create `apps/demo/src/node-details.ts` as the canonical adapter from topology nodes to live detail records.

    The module should define a detail shape that can support both collapsed summaries and the later inspector. Each record should include at least:
    - stable node id
    - node role/type
    - title/label
    - compact summary fields
    - whether drill-down is supported
    - placeholders or slots for recent activity/history

    Use topology node identity from `apps/demo/src/topology.ts` and enrich it from current host/runtime truth in `apps/demo/src/shell-host.ts`. Ensure the adapter covers all currently expected roles:
    - napplet nodes
    - shell node
    - ACL node
    - runtime node
    - service nodes

    Keep the detail model data-first. Do not bake HTML strings into the adapter.
  </action>
