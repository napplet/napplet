# Plan 31-02: NIP-46 Client and Connect Modal

## Objective

Implement a minimal NIP-46 (Nostr Connect / bunker) WebSocket client and the connect modal that presents NIP-07 and NIP-46 side by side as equal first-class connection options.

After this plan:
- Users can open a signer connect modal from the signer node.
- NIP-07 and NIP-46 are presented with equal visual emphasis.
- Users can paste a bunker URI or scan a QR code to connect NIP-46.
- The NIP-46 relay is editable inside the modal only.
- Successful NIP-46 connection wires the bunker signer into the demo signer service.

## Tasks

### Task 1: Create `apps/demo/src/nip46-client.ts`

Implement a minimal NIP-46 requester client. The client handles:

1. An ephemeral local keypair (requester side).
2. A WebSocket connection to the user-specified relay.
3. NIP-46 encrypted request/response framing using nostr-tools `nip04`.
4. A `connect()` handshake call.
5. Per-request `signEvent()` / `getPublicKey()` calls with correlation IDs and timeouts.

**Interface:**

```ts
export interface Nip46ClientOptions {
  relayUrl: string;
  bunkerPubkey: string;
  secret?: string;      // from bunker URI
  timeout?: number;     // ms, default 30000
}

export interface Nip46Client {
  /** Connect to the bunker and complete handshake. Returns the authorized pubkey. */
  connect(): Promise<string>;
  /** Returns a RuntimeSigner-compatible adapter over the bunker connection. */
  getSigner(): RuntimeSigner;
  /** Close the WebSocket and clean up. */
  close(): void;
}

export function createNip46Client(options: Nip46ClientOptions): Nip46Client;
```

**Implementation notes:**
- Use `generateSecretKey()` / `getPublicKey()` from `nostr-tools/pure` for the ephemeral requester keypair.
- Use `nip04.encrypt()` / `nip04.decrypt()` from `nostr-tools/nip04` for message framing.
- Use `finalizeEvent()` from `nostr-tools/pure` to sign request events.
- Wrap the underlying WebSocket in a simple promise-based request/response map keyed on correlation ID.
- `connect()` sends `["connect", bunkerPubkey, secret]` and awaits the `"connect"` response.
- `signEvent()` sends `["sign_event", eventJSON]` and awaits the signed event response.
- `getPublicKey()` sends `["get_public_key"]` and awaits the pubkey string response.
- On timeout, reject the promise with a descriptive error message.

**Parsing bunker URI:**

```ts
export function parseBunkerUri(uri: string): { pubkey: string; relay: string; secret?: string } | null;
```

Handles:
- `bunker://<pubkey>?relay=<url>&secret=<token>`
- `nostrconnect://<pubkey>?relay=<url>` (outbound QR form — different direction)

Return `null` for invalid/unparseable URIs.

### Task 3: Create `apps/demo/src/signer-modal.ts`

Wire modal open/close, NIP-07 connect button, NIP-46 connect button, and QR generation:

```ts
export function initSignerModal(): void;
export function openSignerModal(): void;
export function closeSignerModal(): void;
```

**`initSignerModal()`:**
- Attach click handler to `[data-action="close-signer-modal"]` and backdrop.
- Attach click handler to `[data-action="open-signer-connect"]` anywhere in the document.
- Subscribe to `onStateChange()` from `signer-connection.ts` to update status fields in both columns.
- On modal open: generate and render the QR code for `nostrconnect://` URI (see Task 4).
- On relay field change: regenerate QR.

**NIP-07 connect path:**
- Click on `[data-action="connect-nip07"]`:
  - Disable button, set status to "Connecting...".
  - Call `connectNip07()` from `signer-connection.ts`.
  - On success: show pubkey, close modal after 1.5s.
  - On error: show error text in `#nip07-status`.

**NIP-46 connect path:**
- Click on `[data-action="connect-nip46"]`:
  - Read relay from `#nip46-relay-input` and bunker URI from `#nip46-bunker-uri-input`.
  - Validate that bunker URI is parseable via `parseBunkerUri()`.
  - Override relay from input field if different from URI's relay (relay field takes precedence).
  - Call `connectNip46({ relayUrl, bunkerPubkey, secret })` from `signer-connection.ts` (see Task 5).
  - Disable button, set status to "Connecting to relay...".
  - On success: show pubkey, show active relay, close modal after 1.5s.
  - On error: show error text in `#nip46-status`.

### Task 4: QR Code Generation

Use a lightweight zero-dependency QR code library suitable for ESM: `qrcode` (npm package `qrcode`, browser-compatible, available as `import QRCode from 'qrcode'`) or the inline `qr-creator` package. Both are small (<10KB gzipped).

Generate the `nostrconnect://` URI:

```ts
function buildNostrConnectUri(relayUrl: string, localPubkey: string): string {
  const params = new URLSearchParams({ relay: relayUrl });
  return `nostrconnect://${localPubkey}?${params.toString()}`;
}
```

Render the QR as a canvas or SVG into `#nip46-qr-container`. Update the relay note `#nip46-qr-relay-note` with the relay URL.

If QR rendering fails (e.g., library import error), display the URI as text as a fallback.

### Task 5: Add `connectNip46()` to `signer-connection.ts`

Extend `signer-connection.ts` from Plan 31-01 with:

```ts
export interface Nip46ConnectOptions {
  relayUrl: string;
  bunkerPubkey: string;
  secret?: string;
}

export async function connectNip46(options: Nip46ConnectOptions): Promise<void>;
```

Implementation:
1. Set `isConnecting: true`.
2. Create a `Nip46Client` via `createNip46Client(options)`.
3. Call `client.connect()` to get the authorized pubkey.
4. Set `_activeSigner = client.getSigner()`.
5. Store `_nip46Client = client` (module-level ref for cleanup in `disconnectSigner()`).
6. Update state: `method: 'nip46', pubkey, relay: options.relayUrl, isConnecting: false`.

On error: set `error` message, `isConnecting: false`.

`disconnectSigner()` should call `_nip46Client?.close()` and reset `_nip46Client = null`.

### Task 6: Wire Modal Open from Signer Node "Connect Signer" Button

The signer node button added in Plan 31-01 uses `data-action="open-signer-connect"`. Plan 31-01 used a direct `connectNip07()` call as a temporary measure. This plan replaces that with `openSignerModal()`:

In `main.ts`, update the click handler added in Plan 31-01:

```ts
import { openSignerModal } from './signer-modal.js';

document.addEventListener('click', (e) => {
  const target = e.target as HTMLElement;
  if (target.closest('[data-action="open-signer-connect"]')) {
    openSignerModal();
  }
});

// Remove the old direct connectNip07() call
```

Initialize the modal after the topology renders:

```ts
import { initSignerModal } from './signer-modal.js';

```

## Acceptance Criteria
