    1. Update the `flashEdge` and `flashNode` type signatures to accept `'active' | 'amber' | 'blocked'` instead of `'active' | 'blocked'`.

    2. In the `tap.onMessage` callback (currently around line 140-145), replace the two-state classification with three-state logic:

    ```typescript
    const isOkFalse = msg.verb === 'OK' && msg.raw?.[2] === false;
    const isClosedDenied = msg.verb === 'CLOSED' && typeof msg.raw?.[2] === 'string' &&
      (msg.raw[2].includes('denied') || msg.raw[2].startsWith('blocked:'));
    const isBlocked = isOkFalse || isClosedDenied;

    // Amber: infrastructure failures — not explicit ACL denials, but expected
    // demo-environment failures (no signer, relay stub, timeout, not wired).
    const isAmber = isOkFalse && typeof msg.raw?.[3] === 'string' && (
      msg.raw[3].includes('no signer') ||
      msg.raw[3].includes('relay') ||
      msg.raw[3].includes('timeout') ||
      msg.raw[3].includes('not wired') ||
      msg.raw[3].includes('mock')
    );

    const cls: 'active' | 'amber' | 'blocked' = isAmber ? 'amber' : isBlocked ? 'blocked' : 'active';
    ```

    The OK verb sends: `['OK', event_id, ok_bool, reason_string]` — so `msg.raw[3]` is the reason string.
    The CLOSED verb sends: `['CLOSED', sub_id, reason_string]` — so `msg.raw[2]` is the reason.
    The `isAmber` check only applies to `isOkFalse` messages (OK:false with an infrastructure reason string).
    Explicit ACL denials use `isClosedDenied` and map to `blocked` regardless.

    3. Update the counters block to track amber separately. Replace:
    ```typescript
    if (isBlocked) counters[msg.verb].blocked++;
    ```
    with:
    ```typescript
    if (isAmber) counters[msg.verb].blocked++;  // amber goes in blocked counter bucket (still a failure)
    else if (isBlocked) counters[msg.verb].blocked++;
    ```

    Actually, for clarity in the counter display, keep amber in the blocked bucket — the counter is for "failures" not the specific visual state. The visual distinction is enough.

    4. The existing `highlightPath.nodes.forEach((nodeId) => flashNode(nodeId, cls))` and `highlightPath.edges.forEach((edgeId) => flashEdge(edgeId, cls))` calls work unchanged since `cls` now carries the amber value.
