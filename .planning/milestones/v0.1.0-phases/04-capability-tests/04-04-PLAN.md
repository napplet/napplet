  Create `tests/e2e/signer-delegation.spec.ts` following established Phase 3 test patterns.

  **Critical routing knowledge:** Signer requests are sent as `EVENT` messages with kind `29001` (BusKind.SIGNER_REQUEST). The pseudo-relay's handleEvent routes kind 29001 to `handleSignerRequest()`. The signer request event has tags:
  - `['method', 'getPublicKey' | 'signEvent' | 'getRelays' | 'nip04.encrypt' | etc]`
  - `['id', correlationId]`
  - For signEvent: `['event', JSON.stringify(eventToSign)]`

  Responses:
  - OK message: `['OK', eventId, true/false, reason]` — sent for success/failure acknowledgment
  - Signer response event: kind 29002 (BusKind.SIGNER_RESPONSE) delivered via `deliverToSubscriptions()` with tags:
    - `['id', correlationId]`
    - `['method', method]`
    - `['result', JSON.stringify(result)]`

  The signer response event is NOT sent to the source window directly via postMessage. Instead, it's delivered through the subscription system via `deliverToSubscriptions(responseEvent, null)`. The napplet must have an active REQ subscription matching kind 29002 to receive the response.

  **Therefore:** Before making signer requests, the test must create a subscription for kind 29002 from the napplet's windowId:
  ```typescript
  await page.evaluate(([wid]) => window.__createSubscription__(wid, 'signer-sub', [{ kinds: [29002] }]), [windowId]);
  ```

  **Test setup (beforeEach):**

    }, { timeout: 5000 }).toBe(true);
