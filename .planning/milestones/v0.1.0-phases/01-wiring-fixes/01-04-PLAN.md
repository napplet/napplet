<objective>
Fix AUTH race condition by clearing pendingAuthQueue on all rejection paths.

Purpose: Prevent pre-AUTH messages from a failed AUTH attempt from executing under a later successful AUTH. This is a security bug (Pitfall #3) where the pending message queue is only cleared on signature verification failure but not on challenge mismatch, wrong relay tag, timestamp rejection, or wrong event kind.
Output: Updated handleAuth() with queue cleanup on every rejection path and NOTICE messages per CONTEXT.md D-06.
</objective>

  In the `handleAuth()` function (starting at approximately line 131), there are 5 early-return rejection paths. Currently only the signature failure (path 5) clears the pending auth queue. All paths must clear the queue and send a NOTICE about dropped messages.

  First, create a helper function inside `handleAuth()` right after the `sendOkFail` helper (after approximately line 137):

  ```typescript
  function rejectAuth(reason: string): void {
    const queue = pendingAuthQueue.get(windowId);
    const queueSize = queue?.length ?? 0;
    pendingAuthQueue.delete(windowId);
    sendOkFail(reason);
    if (queueSize > 0) {
      sourceWindow.postMessage(['NOTICE', `${queueSize} queued message(s) dropped due to auth failure`], '*');
    }
  }
  ```
