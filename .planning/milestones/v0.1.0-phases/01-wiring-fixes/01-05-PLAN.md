  **Create `tests/e2e/shell-host.html`:**
  A minimal HTML page that:
  1. Imports `createPseudoRelay` from `@napplet/shell` (via module script with alias)
  2. Imports `createMockHooks` from `./mock-hooks.ts`
  3. Creates a pseudo-relay with mock hooks
  4. Creates a sandboxed iframe pointing to `test-napplet.html` with `sandbox="allow-scripts"`
  5. Registers the iframe in the origin registry
  6. Sends an AUTH challenge to the napplet after iframe loads
  7. Exposes test state on `window.__TEST__` for Playwright assertions

  The shell-host must handle the origin registry setup. Read `packages/shell/src/origin-registry.ts` to understand the `register(windowId, origin, iframeWindow)` API.

  ```html
  <!DOCTYPE html>
  <html>
  <head>
    <meta charset="utf-8">
    <title>Shell Host - E2E Test</title>
  </head>
  <body>
    <div id="status">loading</div>
    <script type="module">
      import { createPseudoRelay } from '@napplet/shell';
      import { originRegistry } from '../../packages/shell/src/origin-registry.js';
      import { createMockHooks } from './mock-hooks.ts';

      window.__TEST__ = { authCompleted: false, messageReceived: false, errors: [] };

      try {
        const hooks = createMockHooks();
        const relay = createPseudoRelay(hooks);

        // Listen for messages
        window.addEventListener('message', (event) => {
          try {
            relay.handleMessage(event);
          } catch (err) {
            window.__TEST__.errors.push(String(err));
          }
        });

        // Create sandboxed iframe
        const iframe = document.createElement('iframe');
        iframe.id = 'test-napplet';
        iframe.sandbox.add('allow-scripts');
        iframe.src = '/test-napplet.html';

        iframe.addEventListener('load', () => {
          const windowId = 'test-napp-1';
          originRegistry.register(windowId, '*', iframe.contentWindow);

          // Send AUTH challenge
          relay.sendChallenge(windowId);
          document.getElementById('status').textContent = 'challenge-sent';
        });

        document.body.appendChild(iframe);

        // Track AUTH completion by watching for OK messages
        window.addEventListener('message', (event) => {
          if (!Array.isArray(event.data)) return;
          const [verb, , success] = event.data;
          if (verb === 'OK' && success === true) {
            window.__TEST__.authCompleted = true;
            document.getElementById('status').textContent = 'auth-completed';
          }
        });

      } catch (err) {
        window.__TEST__.errors.push(String(err));
        document.getElementById('status').textContent = 'error: ' + err;
      }
    </script>
  </body>
  </html>
  ```

  **Create `tests/e2e/smoke.spec.ts`:**
  A Playwright test that:
  1. Navigates to shell-host.html
  2. Waits for the iframe to load and AUTH challenge to be sent
  3. Waits for AUTH to complete (window.__TEST__.authCompleted becomes true)
  4. Checks that no errors occurred (window.__TEST__.errors is empty)
  5. Checks that the status indicates auth completion

  ```typescript
  import { test, expect } from '@playwright/test';

  test('AUTH handshake completes between shell and napplet', async ({ page }) => {
    // Capture console errors
    const consoleErrors: string[] = [];
    page.on('console', (msg) => {
      if (msg.type() === 'error') consoleErrors.push(msg.text());
    });

    await page.goto('http://localhost:4173/shell-host.html');

    // Wait for AUTH to complete (poll window.__TEST__.authCompleted)
    await expect(async () => {
      const result = await page.evaluate(() => (window as any).__TEST__);
      expect(result.authCompleted).toBe(true);
    }).toPass({ timeout: 10_000 });

    // Check no test errors
    const testState = await page.evaluate(() => (window as any).__TEST__);
    expect(testState.errors).toEqual([]);

    // Check no console errors (allow for expected warnings)
    const criticalErrors = consoleErrors.filter(e =>
      !e.includes('Autofocus') && !e.includes('favicon')
    );
    expect(criticalErrors).toEqual([]);
  });
  ```

  Note: This test is intentionally minimal. It proves the e2e wiring works. More comprehensive protocol tests are Phase 2-3 scope. Per CONTEXT.md D-10, this smoke test is disposable verification, not a seed for the Phase 5 demo.
