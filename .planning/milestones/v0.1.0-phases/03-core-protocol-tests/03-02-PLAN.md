  - Each test uses `buildAuthEvent()` to construct an event with a specific defect
  - Each test injects the AUTH event via `__injectMessage__` and asserts on the OK response via `__TEST_MESSAGES__`

  **AUTH-02 through AUTH-09** follow the pattern:
  1. Load auth-napplet (gets a windowId and triggers a challenge)
  2. Clear messages
  3. Build a defective AUTH event using `buildAuthEvent()` with the challenge from `__getChallenge__`
  4. Inject the defective AUTH via `__injectMessage__(windowId, ['AUTH', event])`
  5. Wait for OK message in `__TEST_MESSAGES__`
  6. Assert OK has `success: false` and `reason` contains the expected string
