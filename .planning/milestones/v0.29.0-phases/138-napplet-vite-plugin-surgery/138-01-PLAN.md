<objective>
Subtractive half of the vite-plugin surgery: remove ALL v0.28.0 strict-CSP production machinery from `packages/vite-plugin/src/index.ts`, DELETE `packages/vite-plugin/src/csp.ts` entirely, update `tsup.config.ts` to drop the csp entry, and replace the removed `strictCsp` option field with a `@deprecated` accept-but-warn shim that emits a one-shot `console.warn` per build so existing v0.28.0 consumers do not break on upgrade.

Purpose: Shell is now sole CSP authority in v0.29.0. The plugin stops emitting meta CSP in production (it was only ever a browser-enforcement redundancy — the shell's HTTP-response CSP is authoritative), stops asserting CSP invariants at closeBundle, and removes all CSP-related state. The `strictCsp` option itself stays accepted-but-warned for one release cycle per locked decision Q3 in 138-CONTEXT.md; hard-remove is deferred to v0.30.0 (REMOVE-STRICTCSP).

Output: A leaner `index.ts` with ~250 LOC of CSP machinery gone, a deleted `csp.ts`, a passing build and type-check, and a loud-but-non-breaking deprecation path for napplet authors with `strictCsp: true` in their existing `vite.config.ts`.
