<objective>
Add strict-CSP enforcement to `@napplet/vite-plugin`. Ship a hand-rolled `CspBuilder` helper (`packages/vite-plugin/src/csp.ts`) plus the integration in `packages/vite-plugin/src/index.ts` (extend `Nip5aManifestOptions`, add `transformIndexHtml` with `enforce: 'pre'`, post-build HTML assertions in `closeBundle`).

The plan delivers four project-killer mitigations as build-time hard failures: (1) CSP meta is the literal first `<head>` child, (2) header-only directives (`frame-ancestors`/`sandbox`/`report-uri`/`report-to`) are rejected from meta-delivered CSP, (3) dev-mode HMR `ws://` relaxations cannot leak into production manifests, and (4) `'unsafe-inline'`/`'unsafe-eval'` are forbidden in `script-src`. The default 10-directive baseline locked in CONTEXT.md is emitted verbatim. CAP-03 (`perm:strict-csp` capability) is closed via JSDoc only — no runtime code, since the capability is a shell-side advertisement.

Purpose: This is the load-bearing security phase of v0.28.0. The whole milestone's "browser-enforced isolation" claim depends on the four project-killer pitfalls (Pitfalls 1, 2, 18, 19 from PITFALLS.md) being mechanically impossible to ship. After this phase, a napplet built with `strictCsp: true` ships under a CSP the browser actually enforces, and any developer error attempting to weaken the security posture fails the build with a clear diagnostic.

    5. **`buildBaselineCsp(mode: 'dev' | 'prod', nonce: string, options?: StrictCspOptions): string`** — the canonical CSP string builder. Returns a single-line CSP value (directives joined with `; `, no trailing semicolon, suitable for the `content` attribute of `<meta http-equiv>`). Implementation rules:
       - Compute `connect-src` value: `'none'` for `mode='prod'`; `'self' ws://localhost:* wss://localhost:*` for `mode='dev'`. (CSP-05)
       - Compute `script-src` value: `` `'nonce-${nonce}' 'self'` `` — never anything else baseline. (CSP-07, Pitfall 19)
       - Use `BASELINE_DIRECTIVE_ORDER` to iterate directives in the LOCKED order; for each: pick the baseline value from a literal `baseline` lookup table inside the function, then if `options.directives?.[directive]` is set APPEND those source expressions (separator: single space) to the baseline value. The append-only contract enforces "extend not relax": `connect-src` appending `https://api.example.com` is allowed; appending `'self'` to a baseline of `'none'` is NOT actually allowed because the result `'none' 'self'` is invalid CSP — but document this as "user error caught by the browser".
