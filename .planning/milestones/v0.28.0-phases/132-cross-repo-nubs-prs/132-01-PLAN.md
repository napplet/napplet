    **5. Scheme Protocol Surfaces** (4 subsections, one per scheme ‚Äî covers SCH-02, SCH-03, SCH-04 and references SCH-01):

    `### data:` ‚Äî RFC 2397 per normal. The napplet shim MAY decode `data:` URLs locally without a shell round-trip; conformant shells that receive a `resource.bytes` envelope with a `data:` URL MUST also handle it (decode and return the Blob with the data URL's declared MIME, byte-sniff-validated). No network access. No policy applies beyond the size cap on the decoded Blob.

    `### https:` ‚Äî Shell performs the network fetch. Subject to the full Default Resource Policy below: private-IP block list MUST at DNS-resolution time, MIME byte-sniffing MUST, redirect chain cap SHOULD, response size cap SHOULD, fetch timeout SHOULD, per-napplet rate limit SHOULD. The `mime` returned to the napplet is the shell's byte-sniffed classification ‚Äî never the upstream `Content-Type`.

    `### blossom:` ‚Äî Blossom hash ‚Üí bytes resolution. Canonical URL form: `blossom:sha256:<hex>` where `<hex>` is the lowercase 64-character SHA-256 hex digest. Shells MAY consult any number of Blossom servers per their server-selection policy. The shell MUST verify the returned bytes hash against the URL's declared digest before delivering to the napplet; mismatch MUST result in `code: "decode-failed"`. The `mime` is byte-sniffed by the shell after hash verification.

    `### nostr:` ‚Äî NIP-19 bech32 input form (e.g., `nostr:nprofile1...`, `nostr:naddr1...`). **Single-hop resolution semantics:** the shell resolves the bech32 to its referenced Nostr resource (profile metadata, addressable event content) and returns those bytes as a Blob with `mime: "application/json"` (or another shell-classified MIME if the resource is byte-typed). Shells MUST NOT recursively follow URLs *within* the resolved bytes ‚Äî that is the napplet's job via subsequent `resource.bytes()` calls. Single-hop semantics prevent shell-side fan-out attacks.

    Document scheme dispatch as a whitelist (per Pitfall 6 prevention bullet 3): shells MUST NOT support `file://`, `gopher://`, `dict://`, or other smuggling-prone schemes. Unknown schemes MUST result in `code: "unsupported-scheme"`.

    **6. Default Resource Policy** (covers POL-01 through POL-06; Pitfall 6 anchor):

    Open with: "The shell is the irreducible attack surface for the resource NUB. Conformant shells MUST enforce the following defaults; deviations are a deployment policy decision and SHOULD be surfaced in the shell's documentation."

    `### Private-IP block list (MUST, at DNS-resolution time)` ‚Äî POL-01 ‚Äî verbatim list:
    Shells implementing the `https:` scheme handler MUST, **at DNS-resolution time** (not URL-parse time), reject URLs whose resolved IP falls in any of the following ranges. The check MUST happen after the DNS resolver returns an address and BEFORE the HTTP connection is opened (DNS pinning); each redirect MUST be re-validated independently.

    | Range | RFC / source |
    |-------|--------------|
    | `10/8`, `172.16/12`, `192.168/16` | RFC1918 private |
    | `127/8`, `::1` | Loopback |
    | `169.254/16`, `fe80::/10` | Link-local |
    | `fc00::/7` | Unique-local |
    | `169.254.169.254` | Cloud metadata service (AWS, GCP, Azure, DigitalOcean) |

    Rejection MUST emit `code: "blocked-by-policy"`. Shells MAY allow additional addresses behind explicit shell-administrator policy (e.g. enterprise on-prem services), but the default for community-deployed shells MUST be restrictive.

    `### MIME byte-sniffing (MUST)` ‚Äî POL-05:
    Shells MUST classify the returned bytes via a byte-sniffing strategy (e.g., the [WHATWG MIME Sniffing Standard](https://mimesniff.spec.whatwg.org/) or equivalent). The upstream `Content-Type` header MUST NOT be passed through to the napplet ‚Äî that header is attacker-controlled and the entire trust model breaks if a napplet renders `text/html` because an attacker said so. Shells MUST enforce a scheme-appropriate MIME allowlist (e.g., `https:` for image rendering ‚Äî `image/png`, `image/jpeg`, `image/webp`, `image/gif`, `image/svg+xml` subject to SVG handling below; rejected MIMEs return `code: "blocked-by-policy"`).

    `### Response size cap (SHOULD, recommended ~10 MiB)` ‚Äî POL-02:
    Shells SHOULD cap individual response bodies at a reasonable upper bound; recommended default is 10 MiB. Excess MUST result in `code: "too-large"`.

    `### Fetch timeout (SHOULD, recommended ~30s)` ‚Äî POL-03:
    Shells SHOULD enforce a per-URL fetch timeout; recommended default is 30s. Timeout MUST result in `code: "timeout"`.

    `### Per-napplet concurrency + rate limit (SHOULD)` ‚Äî POL-04:
    Shells SHOULD enforce a per-napplet limit on concurrent in-flight `resource.bytes` calls (recommended: 10) and a per-napplet rate limit (recommended: 60 calls/minute). Rate-limit rejections MUST emit `code: "blocked-by-policy"`.

    `### Redirect chain cap (SHOULD, recommended ‚â§ 5)` ‚Äî POL-06:
    Shells SHOULD cap the redirect chain at 5 hops. Each hop MUST be re-validated against the private-IP block list (DNS pinning per hop). Excess hops or a redirect to a blocked address MUST result in `code: "blocked-by-policy"`.

    **7. SVG Handling** (covers SVG-01, SVG-02, SVG-03; Pitfall 7 anchor):

    `### Shell-side rasterization (MUST)` ‚Äî SVG-01, SVG-02:
    When the shell's MIME byte-sniffer identifies a fetched resource as `image/svg+xml`, the shell MUST rasterize it to a bitmap format (PNG or WebP) before delivering the result envelope to the napplet. Shells MUST NOT deliver raw `image/svg+xml` bytes to napplets. The `mime` field on the result envelope MUST be `image/png` or `image/webp` for any SVG-source-input result.

    Rationale: SVG `<foreignObject>`, `<script>`, `<image href>`, `<use>` recursion, and XML DOCTYPE entity expansion are documented attack surfaces (see Security Considerations ¬ß "SVG attack surface"). Rasterization to a flat bitmap eliminates the entire class.

    `### Rasterization isolation (MUST)` ‚Äî SVG-03 (Pitfall 7 anchor):
    Rasterizers MUST run in a **sandboxed Worker** with **no network** access ‚Äî the rasterizer MUST NOT issue any network request, including SVG-internal `<image href>`, `@font-face` `src` URLs, `<use href>` external references, or DOCTYPE entity URLs. SVG-internal references MUST resolve to local-only data or be omitted from the rasterized output.

    `### Rasterization caps (SHOULD)` ‚Äî SVG-03:
    | Cap | Recommended default | On exceed |
    |-----|---------------------|-----------|
    | Max input bytes | 5 MiB | `code: "too-large"` |
    | Max output dimensions | 4096 √ó 4096 pixels | `code: "too-large"` |
    | Wall-clock rasterization budget | 2 s | `code: "timeout"` |

    Document the billion-laughs / `<foreignObject>` / recursive `<use>` attack patterns briefly so the cap rationale is self-evident.
